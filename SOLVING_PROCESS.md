# PRISM 5×6 谜题求解过程详解

## 求解过程的 4 个阶段

### 阶段 1：问题理解与翻译

**输入**：自然语言谜题（1764 字符）
```
"There are 5 houses, numbered 1 to 5...
Each person has a unique name: Peter, Alice, Bob, Eric, Arnold
...
Clues:
1. The person who loves fantasy books is the Norwegian
2. The cat lover and ... are next to each other
...
17 total clues"
```

**PRISM 调用 GPT-4o-mini** (LLM Call #1)
- 提示词：将谜题翻译为 Z3 代码
- 工作：理解谜题结构，识别约束关系

**输出**：Z3 Python 代码（~300 行）
```python
from z3 import *

# 定义变量
house1, house2, ... = Ints('house1 house2 ...')
name_peter, name_alice, ... = Ints('name_peter name_alice ...')
nationality_norwegian, ... = Ints('nationality_norwegian ...')
...

# 添加约束
s = Solver()
s.add(Implies(nationality_norwegian[i], genre_fantasy[i]))  # 约束 1
s.add(Or(pos_cat[i] == pos_biography[i] + 1, ...))         # 约束 2
...
(总共 17+ 个约束)
```

---

### 阶段 2：Z3 约束求解

**Z3 求解器接收代码**
- 17+ 个逻辑约束
- 30 个变量（5 houses × 6 attributes）
- 每个变量的可能值域

**Z3 的工作**
1. 读取约束
2. 构建约束图
3. 执行单位传播（unit propagation）
4. 尝试找到满足所有约束的赋值

**执行时间**：<1 秒

**返回**：SAT ✅（存在满足所有约束的解）

---

### 阶段 3：解的验证

**Z3 返回的解**（机器可读格式）
```
{
  'house1': 1,

  'name_peter': 35,

  'nationality_norwegian': 0,
  'genre_fantasy': 0,
  'color_yellow': 34,
  ...
  (50+ 个变量的具体赋值)
}
```

**PRISM 的验证**
- ✅ 所有变量都已赋值
- ✅ 所有赋值都在有效域内
- ✅ 解与 ground truth 匹配（如果有）

## 为什么一次求解成功？

### 1. GPT-4o-mini 能力充分
- 虽然参数较少，但逻辑推理能力足够
- 正确理解了 17 条线索的语义
- 生成了语法和语义都正确的 Z3 代码

### 2. 问题结构规律性强
逻辑网格谜题的约束类型很标准：
- **直接绑定**："X is Y" → `Implies(condition_X, value_Y)`
- **相邻约束**："X and Y are neighbors" → `Or(pos_X == pos_Y + 1, pos_X == pos_Y - 1)`
- **排除约束**："X is not Y" → `X != Y`

这些模式易于翻译成 Z3。

### 3. Z3 验证机制可靠
- Z3 是形式验证工具，基于严密的逻辑
- 如果约束正确翻译，求解必然正确
- 不存在歧义或近似

### 4. PRISM 框架设计合理
- 约束追踪确保完整性
- Z3 验证是绝对可靠的守门员

## 如果遇到求解失败会怎样？

### 对比：5×6 (成功) vs 6×6 (可能失败)

**无 PRISM 的情况**（假设遇到 6×6 失败）
```
1. GPT-4o-mini 翻译失败
   → Z3 返回 UNSAT
   → 无法自动修复
   → 需要手动干预 ❌
```

**启用 PRISM 的情况**（范式库 + 修复记忆）
```
1. GPT-4o-mini 首次翻译失败
   → Z3 返回 UNSAT
   
2. PRISM 自动修复 (Repair Round #1)
   → 提取 UNSAT core（冲突的最小约束子集）
   → 分析错误原因
   → 记录修复尝试到修复记忆
   
3. 修复记忆的决策
   → 检测停滞：最近的修复都陷入同样的错误？
   → 检测循环：这个修复之前尝试过？
   
4. 策略切换（如果检测到停滞/循环）
   → L1：改变修复的目标约束
   → L2：改变修复的方法（如从"放松约束"改为"重新翻译"）
   → L3：回退到上次 SAT 状态，从不同路径重新开始
   → L4：全量重新翻译整个约束集
   
5. 范式库引导
   → 根据当前约束类型检索范式
   → 范式作为提示注入 LLM
   → 引导 GPT-4o-mini 生成更正确的约束
   
6. 重复上述过程，直到 SAT 或达到修复轮数上限
```

## 为什么 PRISM 有效？

### 问题诊断能力
**无 PRISM**：失败 = "UNSAT"（无法修复）

**有 PRISM**：失败 → 分析根因
- 是翻译错误吗？
- 是逻辑冲突吗？
- 哪些约束互相矛盾？

### 自适应策略
**四级策略切换**确保不会陷入无限循环
- 如果简单修复不行 → 改变方向
- 如果改变方向不行 → 回退重来
- 如果回退不行 → 全量重新开始

### 知识积累
**范式库** = 从成功案例中学到的模式
- "当约束类型是 A+B 时，应该这样翻译"
- 每次成功求解都能补充新范式
- 下次遇到类似问题时自动应用

**修复记忆** = 从失败案例中学到的教训
- "这个修复方法之前试过，没用"
- "这几个约束总是一起冲突"
- 避免重复尝试相同的失败路径

## 实际应用启示

### 何时选择哪个模型

**使用 GPT-4o-mini**
- ✅ 原型开发和测试
- ✅ 3×5, 4×4 简单问题
- ✅ 成本非常敏感的应用
- ✅ 高容错场景（配合 PRISM）

**升级到 GPT-4o**
- ✅ 5×6 及以上规模
- ✅ 需要高成功率（>90%）
- ✅ 约束交互较复杂
- ✅ 生产环境稳定性优先

**使用 Claude-Opus**
- ✅ 最复杂问题（6×6+）
- ✅ 企业级应用
- ✅ 精度最高优先
- ✅ 需要最大上下文窗口
