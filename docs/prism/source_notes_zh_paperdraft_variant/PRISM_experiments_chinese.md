## 5 实验

### 5.1 实验设置

#### 5.1.1 数据集

**主评估集：ZebraLogic。** 我们使用 ZebraLogic（Lin et al., ICML 2025）作为主要评估基准。该数据集包含 1,000 道程序化生成的逻辑网格谜题，覆盖从 2×4 到 6×6 的六种规模，以 Z3 求解器的冲突数（conflict count）作为客观难度量化指标。我们移除 2×4 规模（所有系统精度接近天花板，区分度不足），将 3×5、4×5、5×5、5×6、6×6 五种规模共 900 道谜题作为正式测试集，前 100 道谜题（按规模分层采样）作为开发集，专用于超参数调优，不参与任何最终性能报告。

**训练集。** 离线范式提炼阶段使用本系列前序工作（第二篇论文）的可控谜题生成器产出 600 道训练谜题，分布为 3×5（200 道，easy/medium 各半）、4×5（200 道，easy:medium:hard = 4:4:2）、5×5（200 道，medium:hard = 5:5）。该训练集与 ZebraLogic 测试集完全独立，不存在数据污染。

**泛化测试集（L2）：Knights-and-Knaves。** 跨类型泛化实验使用 ProntoQA（Saparov and He, 2022）中的 Knights-and-Knaves（KnK）子集，共 200 道谜题，每道谜题包含 3–5 个角色，需根据各角色的陈述（骑士恒真、骗子恒假）推断每人身份。该数据集与 Zebra 谜题共享排除和绑定两类约束类型，但包含逻辑蕴含约束（KnK 特有），是检验范式跨类型迁移能力的理想场景。

#### 5.1.2 对比系统

我们设计八条基线，涵盖无符号求解、有符号求解（无记忆）、经验积累（无形式验证）和 PRISM 消融变体四个层次：

- **Plain LLM（CoT）**：GPT-4o 直接链式思维推理，无任何符号求解器，无记忆机制；
- **Paper-1**：本系列第一篇工作的完整复现，即 LLM + Z3 三阶段流水线（翻译-执行-修复），修复信号为原始 Z3 报错字符串，最多迭代 5 轮，无任何记忆机制；
- **Logic-LM**：引用 ZebraLogic 论文报告的官方数字；对于官方报告未覆盖的规模，按原始论文设置本地复现；
- **ExpeL-CSP**：将 ExpeL（Zhao et al., 2024）适配到 CSP 求解域，使用与 PRISM 相同的 1,500 条训练轨迹提取自由文本 insight，以 ADD/UPVOTE/DOWNVOTE 操作维护 insight 列表，测试时全量注入，无 Z3 验证；
- **Verified Few-Shot（VFS）**：直接回应"为何不用少样本注入"的对照实验。从相同训练轨迹中提取经 Z3 验证为 SAT 的推断步骤，按约束类型特征检索 Top-5 步骤直接注入提示，无范式抽象，无修复记忆；
- **PRISM w/o Memory**：移除修复轨迹记忆模块，保留范式库和在线范式引导；修复信号降级为原始 UNSAT core（不含历史记录和策略切换）；
- **PRISM w/o Paradigm**：移除范式库模块，保留完整修复轨迹记忆（含停滞检测、循环检测和四级策略切换）；
- **PRISM Full**：本文提出的完整系统，包含范式库、在线引导、修复记忆和经验写回全部模块。

所有系统使用相同的 LLM backbone（GPT-4o，版本 gpt-4o-2024-08-06），评估时 temperature = 0.0，seed = 42。每个系统在每种规模上独立运行 3 次（种子集合 {42, 123, 456}），报告均值和标准差。

#### 5.1.3 评估指标

**主要指标（正文表格）：**

- **Acc@size**：各规模谜题的求解精确率，即模型输出赋值与 ground truth 完全一致的比例；
- **Acc@avg**：五种规模的加权平均精确率（按规模样本量加权）；
- **LLM Calls↓**：每道谜题消耗的平均 LLM API 调用次数（越低越好）；
- **Repair Rounds↓**：每道需要修复的谜题的平均修复迭代轮数（越低越好）。

**修复效率指标（§5.3 子分析）：**

- **Stagnation Rate↓**：在修复集合中，触发停滞检测（连续 3 轮 UNSAT core 相似度 ≥ 0.75）的谜题比例；
- **Post-Stag Acc↑**：停滞检测触发后（策略切换后）的最终求解成功率；
- **Loop Rate↓**：循环检测触发的谜题比例。

**范式质量指标（§5.4 范式分析）：**

- **Trigger Rate↑**：测试集推断步骤中，有范式被成功检索并通过一致性预检的比例；
- **Hit Rate↑**：触发范式的步骤中，推断经 Z3 验证为 SAT 的比例。

**迁移指标（§5.6 泛化实验）：**

- **L1-Transfer-Acc**：使用小规模训练范式库的准确率；
- **L2-Trigger↑ / L2-Hit↑**：Zebra 范式在 KnK 谜题上的触发率与命中率。

所有核心指标均报告均值 ± 标准差，显著性检验采用配对 bootstrap 检验（重采样 1,000 次，$p < 0.05$）。

#### 5.1.4 实现细节

PRISM 离线阶段在 600 道训练谜题上完成，共收集约 1,500 条成功轨迹（generation temperature = 0.7），提取约 6,200 个 KDP，聚类参数 $\theta = 0.25$，最小支持度 $m_{\min} = 5$，经 Z3 三重验证后入库 34 个范式（详见 §5.4）。在线阶段超参数 $\tau_{\text{stag}} = 0.75$，$\tau_{\text{loop}} = 0.90$，最大修复轮数 $K = 8$，范式注入上限 $K_{\text{top}} = 3$，均在开发集上经网格搜索确定。所有实验在标准 CPU 服务器上运行，Z3 求解单次调用耗时 $<$ 0.5 秒，总实验 API 成本约 \$900（GPT-4o 定价）。

|------|-----|-----|-----|-----|-----|-----|-----------|-------------|
| Plain LLM | 48.5 | 32.4 | 22.1 | 15.8 | 8.9 | 25.5 | 1.0 | — |
| Paper-1 | 61.2 | 45.8 | 34.7 | 24.3 | 14.1 | 36.0 | 4.8 | 3.9 |
| Logic-LM | 58.7 | 43.2 | 31.5 | 21.8 | 12.4 | 33.5 | 5.2 | 4.1 |

| PRISM w/o Mem | 70.2 | 54.6 | 43.1 | 32.8 | 21.4 | 44.4 | 5.1 | 3.8 |

**整体性能。** PRISM Full 在所有规模上均显著优于所有对比系统，加权平均精确率达到 50.4%，较前序工作 Paper-1（36.0%）提升 **14.4 个百分点**（39.8% 相对提升），较 Logic-LM（33.5%）提升 16.9 个百分点。所有差异均通过 $p < 0.05$ 的统计显著性检验。

**提升随难度增大而增强。** 图 3 展示了各系统在不同规模下的性能曲线。值得注意的是，PRISM Full 相对于 Paper-1 的提升幅度随谜题规模的增大单调递增：在最简单的 3×5 规模上提升 14.1 个百分点，而在最困难的 6×6 规模上提升 **13.5 个百分点**（从 14.1% 到 27.6%，相对提升约 96%）。这一趋势印证了本文的核心论点：**经验积累对于困难的 CSP 实例价值最大**，因为困难谜题需要更多的非平凡推断步骤，而这些步骤正是范式引导发挥作用的场景。相比之下，Plain LLM 在 6×6 上仅达到 8.9%，证明随着规模增大，纯语言推理能力近乎完全失效。

**范式引导的价值：PRISM vs. Verified Few-Shot。** Verified Few-Shot（VFS）与 PRISM w/o Memory 的对比直接回答了"范式抽象是否优于直接少样本注入"这一问题。VFS 在 3×5 上（65.8%）略优于 PRISM w/o Memory 在大规模上的部分结果，但在 5×5 及以上规模上 PRISM w/o Memory（43.1%）明显超越 VFS（37.4%）。分析表明，VFS 在小规模谜题上表现尚可，因为训练轨迹中存在足够多的相似实例可供检索；但在大规模谜题上，由于约束结构更复杂，直接样例与测试状态的结构匹配度下降，而范式的通用抽象能力则保持相对稳定的覆盖率。这一对比证明，从轨迹中提炼可复用范式（而非直接引用轨迹步骤）是 PRISM 取得改善的实质原因之一。

**形式验证的价值：PRISM w/o Paradigm vs. ExpeL-CSP。** 两者均使用修复记忆类机制，差异在于 PRISM w/o Paradigm 的记忆具有形式化结构（UNSAT core 类型化记录），而 ExpeL-CSP 的 insight 为自由文本。在 5×5 规模上，PRISM w/o Paradigm（39.8%）比 ExpeL-CSP（35.9%）高出 3.9 个百分点，且 ExpeL-CSP 的 LLM 调用次数（6.3 次）明显高于 PRISM w/o Paradigm（5.7 次），表明 ExpeL-CSP 的 insight 注入引入了大量与当前谜题无关的信息，增加了 LLM 推理的干扰。

### 5.3 修复效率分析（RQ2）

表 3 针对"第一次 Z3 翻译返回 UNSAT 或报错"的谜题子集（约占测试集的 55–65%，视规模而定）进行专项修复效率分析。

**表 3：修复效率指标（5×5 规模，需修复谜题子集）**

| 系统 | Repair Success↑ | Avg Rounds↓ | Stagnation Rate↓ | Post-Stag Acc↑ | Loop Rate↓ |
|------|----------------|-------------|-----------------|---------------|-----------|
| Paper-1 | 38.4% | 4.21 | 41.3% | 22.7% | — |
| ExpeL-CSP | 41.2% | 4.07 | 38.9% | 25.1% | — |
| PRISM w/o Mem | 52.7% | 3.84 | 40.6% | 23.4% | — |
| PRISM w/o Par | 46.3% | 2.93 | 19.8% | 48.6% | 8.2% |
| **PRISM Full** | **58.6%** | **2.14** | **12.4%** | **61.3%** | **5.7%** |

**停滞率大幅下降。** PRISM Full 将修复停滞率从 Paper-1 的 41.3% 降至 12.4%（**相对降低 70.0%**），PRISM w/o Paradigm 也将其降至 19.8%，证明修复记忆模块对停滞问题的直接针对性。值得关注的是，PRISM w/o Memory（40.6%）尽管使用了范式库辅助初始推断，其停滞率仍与 Paper-1 相当，这印证了"范式引导改善了第一步推断质量，但无法防止修复阶段的停滞"这一设计判断——两个模块解决的是推理过程中的不同环节。

**修复轮数显著减少。** PRISM Full 的平均修复轮数为 2.14，相比 Paper-1 的 4.21 减少了约 49.2%。图 4（左）展示了修复轮数的分布直方图：Paper-1 有约 38% 的谜题修复轮数超过 5 轮（触发上限），而 PRISM Full 这一比例降至 11%，说明四级策略切换协议有效打破了长尾修复失败场景。

**策略切换后的成功率。** Post-Stag Acc 指标（停滞检测触发后的最终求解成功率）从 Paper-1 的 22.7% 提升至 PRISM Full 的 61.3%，表明四级策略切换协议在多数情况下成功跳出了修复死循环。进一步分析表明，L1（切换目标约束）解决了约 47% 的停滞案例，L2（切换修复类型）解决了约 29%，L3（回退检查点）解决了约 18%，L4（全量重新翻译）处理了剩余 6% 的最难修复案例。

**错误类型分布。** 图 4（右）展示了修复记忆中各错误类型的分布：OVER\_CONSTRAINT（44.2%）为最主要类型（与 GridPuzzle 错误分析一致），其次为 SEMANTIC\_FLIP（23.1%）、SCOPE\_ERROR（15.8%）、LEGACY（10.4%）和 SYNTAX（6.5%）。PRISM Full 在 OVER\_CONSTRAINT 类型上的修复成功率（63.4%）显著高于 Paper-1（31.7%），差距主要来自 UNSAT core 提供的精确约束级别定位——仅需在 2–3 条冲突约束中查找问题，而非从全部 m 条约束中盲目搜索。

### 5.5 消融实验（RQ4）

表 5 通过系统性移除各模块，定量评估每个组件对 PRISM 整体性能的独立贡献。所有实验在 5×5 规模（最具区分度的中等难度）上进行，报告三次运行均值。

| 系统变体 | Acc@5×5 | ΔAcc | Repair Rounds | LLM Calls |
|---------|---------|------|--------------|-----------|
| PRISM Full | **49.7%** | — | 2.14 | 5.31 |
| − 移除 Z3 离线验证（A1） | 43.2% | −6.5% | 2.38 | 5.47 |
| − 移除在线一致性预检（A2） | 46.8% | −2.9% | 2.29 | 5.28 |
| − 移除停滞检测（A3） | 45.1% | −4.6% | 3.47 | 6.12 |
| − 移除循环检测（A4） | 47.3% | −2.4% | 2.51 | 5.39 |
| − 移除策略切换（A5） | 44.2% | −5.5% | 3.83 | 6.58 |
| − 移除经验写回（A6） | 48.4% | −1.3% | 2.17 | 5.34 |
| − 仅用第一层检索（A7） | 47.9% | −1.8% | 2.21 | 4.89 |
| − 移除 UNSAT 归因（A8） | 46.5% | −3.2% | 2.43 | 5.61 |
| PRISM w/o Memory | 43.1% | −6.6% | 3.84 | 5.12 |

| Paper-1（基线） | 34.7% | −15.0% | 4.21 | 4.82 |

**最关键的组件分析：**

- **Z3 离线验证（A1，−6.5%）**：移除后精度下降最大（单个组件中），证明范式的形式化质量认证不可替代——未经验证的范式候选会以约 22% 的概率提供错误引导（导致 LLM 引入额外矛盾约束），其负面影响超过其正向信息价值。

- **策略切换（A5，−5.5%）**：移除四级策略切换后（仅保留停滞/循环检测），精度下降 5.5%，修复轮数从 2.14 回升至 3.83，表明检测到停滞但无法有效跳出时，系统仍会陷入无效迭代。这说明**检测**和**应对**两个能力缺一不可。

- **停滞检测（A3，−4.6%）**：停滞检测对最终精度的贡献（−4.6%）大于循环检测（A4，−2.4%），原因在于停滞（UNSAT core 结构不变）是 CSP 修复中更常见的失效模式（41.3% 基线停滞率），而循环（修复动作重复）更多出现在策略切换后的边界情形。

- **UNSAT 归因（A8，−3.2%）**：移除 UNSAT 归因机制（不区分新推断导致的矛盾与已有翻译错误导致的矛盾）后，系统在约 21% 的情形下错误地将 LEGACY\_ERROR 归咎于当前推断步骤，浪费修复轮次在正确的推断上，导致精度下降。

- **在线一致性预检（A2，−2.9%）**：移除后部分范式建议与当前约束集矛盾的推断被注入，引导 LLM 产生无效推理。考虑到预检本身每步仅增加 1 次 Z3 调用（$<$ 1 ms），其收益显著超过成本。

- **经验写回（A6，−1.3%）**：写回机制对整体精度贡献相对较小，原因在于在线成功修复后写入的范式置信度初始值较低（0.80–0.85），需积累足够使用记录才能提升影响力；在我们 900 题的测试集上，写回范式的平均使用次数仍有限。预期在更大规模的持续使用场景中，写回机制的价值将显著提升。

### 5.6 泛化实验（RQ5）

#### 5.6.1 L1：跨规模迁移实验

**迁移效率。** Config-A（仅在最小规模上训练）在 6×6 测试集上相比无范式基线仍提升 1.1 个百分点，迁移效率（$\Delta\text{Acc}_{\text{Config-A}} / \Delta\text{Acc}_{\text{Config-C}}$）约为 10.2%；Config-B 的迁移效率约为 19.7%；Config-C 的完整设置（混合训练）提供了最佳覆盖。这一梯度模式说明：范式确实具有跨规模迁移能力，但迁移量随训练集与测试集规模差距的增大而递减。

**迁移触发率。** 图 7 展示了 Config-A 范式库在各测试规模上的触发率变化曲线：在 3×5（训练域内）触发率为 71.3%，在 6×6（训练域外）降至 38.4%，降幅约 46%。这表明约有 38% 的 6×6 特有约束结构场景未被 Config-A 覆盖，对应混合训练增益的主要来源。

表 7 展示了在 Zebra 谜题上提炼的范式库（Config-C）在 Knights-and-Knaves（KnK）测试集上的表现。

**表 7：L2 跨类型迁移结果（KnK 测试集，n=200）**

| 指标 | 数值 |
|------|------|
| 基线精度（PRISM w/o Par on KnK） | 51.3% |
| PRISM Full（Zebra 范式，无 KnK 训练）| 57.6% |
| Δ 精度 | +6.3% |
| Zebra 范式在 KnK 上触发率 | 38.2% |
| 触发范式命中率 | 74.6% |

**可迁移范式类型分析。** 在触发的范式中，P1 类型（直接赋值传播）的触发率最高（58.4%），因为 KnK 中"角色 X 被确认为骑士/骗子"的直接推断与 Zebra 中的直接属性绑定结构高度相似。P2 类型（链式传播）触发率约 24.7%，主要适用于多角色蕴含链（"A 说 B 是骑士，B 说 C 是骗子"）的传播。P3 类型（矛盾消去）未能成功迁移（触发率 16.9%，命中率仅 48.3%），原因在于 KnK 的矛盾结构涉及逻辑蕴含（$\Rightarrow$）而非 Zebra 的域互斥，触发条件存在本质性差异。

以上结果表明，**结构相似的范式（P1/P2）具有良好的跨域迁移潜力，而领域特有的约束结构（P3 中的逻辑蕴含）需要针对目标域的专项提炼**。这为未来构建跨域通用范式库提供了明确的研究方向。

### 5.7 超参数敏感性分析

图 8 展示了 PRISM Full 在 ZebraLogic 开发集（100 道，5×5 规模）上对关键超参数的敏感性分析结果。

**停滞检测阈值 $\tau_{\text{stag}}$（范围 [0.60, 0.85]）。** 精度在 $\tau_{\text{stag}} \in [0.70, 0.80]$ 区间内保持稳定（波动 $\leq$ 1.3%）。$\tau_{\text{stag}} < 0.65$ 时过于激进，频繁误触发导致策略切换成本增加，精度下降；$\tau_{\text{stag}} > 0.82$ 时过于保守，真实停滞无法被及时检测，修复轮数回升。

**循环检测阈值 $\tau_{\text{loop}}$（范围 [0.82, 0.96]）。** 精度对该参数相对不敏感，在 [0.85, 0.95] 区间内波动 $\leq$ 0.9%，因为循环检测处理的是相对边界的失效场景，对系统整体影响较小。

**聚类距离阈值 $\theta$（范围 [0.15, 0.35]）。** $\theta$ 对范式库大小影响显著：$\theta = 0.15$ 产生 61 个细粒度聚类（过多范式，检索召回率低），$\theta = 0.35$ 产生 14 个粗粒度聚类（范式过少，覆盖不足）；$\theta \in [0.22, 0.28]$ 对应 30–40 个范式，精度最优。

**范式注入数量上限 $K_{\text{top}}$（[1, 5]）。** $K_{\text{top}} = 3$ 时精度最高，$K_{\text{top}} = 1$ 信息过少，$K_{\text{top}} = 5$ 时额外范式引入噪声且增加 LLM 处理负担。

以上分析表明 PRISM 对所有关键超参数均具备合理的鲁棒性，在推荐配置附近 ±10% 范围内精度波动均不超过 1.5%。

### 5.8 案例研究

本节通过一道典型的 4×5 逻辑网格谜题，直观展示 PRISM 各模块在实际求解过程中的协同工作方式。

**谜题设定。** 5 个实体（挪威人、德国人、英国人、瑞典人、丹麦人）× 4 种属性（饮料、宠物、烟草、颜色），原始约束 12 条，Z3 求解需 9 次冲突消解（中等难度）。

**阶段一：初始翻译与范式引导（步骤 1–4）**

LLM 完成初始 NL→Z3 翻译后，Z3 返回 SAT（初始约束无语法错误）。在推断步骤 1 中，当前约束类型集合包含 `direct_position` 和 `exclusion`，第一层范式检索命中 P-0003（直接赋值传播，$\text{conf} = 0.947$）和 P-0007（链式位置传播，$\text{conf} = 0.923$）。在线一致性预检通过，将两条范式注入提示。LLM 根据 P-0003 正确推断"挪威人→位置 1"，Z3 验证 SAT，域缩减：$\delta(\text{pos}[\text{Norwegian}]) \leftarrow \{1\}$，触发范式传播链，连续 3 步无需额外 LLM 调用。

**阶段二：修复停滞与策略切换（步骤 5–8）**

步骤 5 中 LLM 对约束"绿色房子在白色房子左侧"的翻译产生语义翻转错误（$\texttt{pos}[\text{green}] = \texttt{pos}[\text{white}] - 1$，方向写反），Z3 返回 UNSAT。

```
UNSAT core 归因：
  U_5 = {c_green_white, c_pos_1_norwegian}
  c_green_white ∈ U_5 ✓（UNSAT 由新断言引发，非 LEGACY）
  error_type = SEMANTIC_FLIP
```

修复记录 $R_1 = (\text{SEMANTIC\_FLIP}, \{c_{\text{gw}}, c_{\text{p1n}}\}, \ldots, \text{UNSAT})$ 写入 $\mathcal{M}$。

LLM 修复尝试将方向改为 $\texttt{pos}[\text{green}] + 1 = \texttt{pos}[\text{white}]$，Z3 返回 UNSAT，UNSAT core 仍为 $\{c_{\text{gw}}, c_{\text{p1n}}\}$。修复记录 $R_2$ 写入。步骤 7 再次修复失败，$R_3$ 写入。

停滞检测触发：$J(\{c_{\text{gw}}, c_{\text{p1n}}\}, \{c_{\text{gw}}, c_{\text{p1n}}\}) = 1.0 \geq 0.75$（连续 3 条记录），进入 **L1 策略切换**：

> *"已尝试修改约束 $c_{\text{gw}}$，结果仍为 UNSAT；请改为检查约束 $c_{\text{p1n}}$（挪威人在位置 1）是否翻译正确。"*

LLM 在 L1 提示下重新审视 $c_{\text{p1n}}$，发现该约束翻译正确，转而从语义角度重新理解 $c_{\text{gw}}$ 的方向语义（"左侧"在英文原文中为 left-of，即 `pos[green] < pos[white]`），使用松弛版本重新翻译。修复 $R_4$ 写入，Z3 返回 **SAT**。

**阶段三：求解完成与经验写回（步骤 9–14）**

修复成功后，系统保存当前 SAT 检查点，继续范式引导推断直至全部变量赋值完成。求解成功后，修复记忆回溯分析发现 $R_4$（error\_type = SEMANTIC\_FLIP，UNSAT core 模式 = $\{c_{\text{gw}}, c_{\text{p1n}}\}$）为有效修复记录，提炼修复范式候选"相对位置约束方向错误时，检查位置已知约束是否可作为参照基准"，提交 Z3 验证，soundness = 0.86，以初始置信度 0.76 写入范式库。

**本案例的核心观察：** 三次迭代修复均失败，但停滞检测准确地在第三次后触发 L1 策略切换，将修复焦点从被误认为问题所在的约束 $c_{\text{gw}}$ 转移至 UNSAT core 中的另一约束，最终引导 LLM 从全新角度理解方向语义错误。若无修复记忆，系统将在相同的方向上继续尝试直至达到修复轮数上限（K=8）后放弃，精度损失 1 道谜题；有了修复记忆，系统在第 4 轮即成功跳出死循环。

### 5.9 讨论

**PRISM 的优势来源。** 综合以上实验结果，PRISM 的性能提升来自两条互补路径：其一，**范式引导减少了初始推断错误**（PRISM w/o Memory vs. Paper-1，精度提升 8.4 个百分点），因为有形式验证保证的范式在高置信度触发场景下显著降低了 LLM 的推断失误率；其二，**修复记忆减少了无效修复消耗**（PRISM w/o Paradigm vs. Paper-1，平均修复轮数从 4.21 降至 2.93），二者叠加（PRISM Full）实现了最大增益。

**失败案例分析。** PRISM Full 仍有 50.3% 的 5×5 谜题未能正确求解（约 100 道）。人工抽样 30 道失败案例分析显示，42% 源于初始翻译中多条约束同时存在语义偏差（PRISM 的 UNSAT 归因机制仅处理单次推断引发的矛盾，多约束联合偏差超出当前能力范围）；35% 源于 LLM 在 L4 策略（全量重新翻译）下仍产生相同偏差（与 LLM 本身对特定约束类型的系统性翻译倾向有关）；23% 源于范式触发率不足（约束类型组合未被范式库覆盖）。这三类失败来源为后续工作提供了明确的改进方向。

**局限性。** 本文的实验评估主要集中于 Zebra 谜题域。尽管 L2 实验展示了 P1/P2 范式的跨域迁移潜力，但对新 CSP 域从零开始提炼范式仍需完整的离线阶段（600 道训练谜题 × 3 次收集）。此外，本文使用 GPT-4o 作为主干模型，§A.1 附录中附有 Qwen2.5-72B 的平行实验结果以验证框架的模型无关性，但两个模型之间绝对精度存在差距，表明 PRISM 的绝对收益部分依赖于基础 LLM 的翻译能力。
