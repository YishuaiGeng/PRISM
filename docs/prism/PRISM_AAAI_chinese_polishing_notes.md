## 已修正的逻辑问题

1. 范式在方法中是五元组，原引言曾写成四元组。规范稿统一为 $(\mathcal{Q},\mathcal{O},\phi_{\mathrm{pre}},\phi_{\mathrm{post}},\mathcal{S})$。
2. Z3 的 SAT 检查只能说明候选操作与当前形式化状态相容，不能证明自然语言翻译正确，也不能凭有限采样证明范式在全部状态上正确。
3. Z3 返回的 UNSAT core 不必是最小核心。除非实际运行了最小化算法，主文只能称其为“求解器返回的冲突核心”。
4. 主结果中相对前序流水线的绝对提升为 14.1、15.0、15.0、14.1、13.5 个百分点，不能写成“绝对提升随难度单调增加”。相对提升可以描述为随规模上升。
5. `PRISM w/o Paradigm` 与 `ExpeL-CSP`、以及 `Verified Few-Shot` 与 `PRISM w/o Memory` 都不是只改变一个因素的严格对照。规范稿将其改为配置级观察，不再将差异单独归因于形式验证或范式抽象。
6. 在线写回目前只有有限使用次数的证据。它应被表述为可审计的持续学习接口，而不是已被主实验充分证实的长期收益。
7. 原始材料将 67.4% 同时描述为“第一层类型匹配”与 `Trigger Rate`，而指标定义又把后者写为通过一致性预检。最终稿必须二选一并重新计算相应指标。
8. 原始段落中的约数与结果表存在漂移，例如 1,500/1,487 条轨迹、6,200/6,218 个 KDP、约 35/34 个入库范式。规范稿统一采用结果表的精确计数，最终仍需以日志为准。

## 投稿前必须核验

- 逐项将表 1 至表 3 与固定版本代码、原始 JSONL/CSV、随机种子和运行日志对齐。当前工程中存在持续开发中的未提交实验代码，不能把草稿数字视为最终结果。
- 以相同主干模型、提示预算、最大修复轮数和数据划分重跑所有本地基线。若 `Logic-LM` 的任何单元格沿用公开报告值，单独标注来源，不做配对显著性比较。
- 在主表或补充材料补齐样本数、均值、标准差/置信区间、配对单位、bootstrap 设置和所有拒绝/超时实例的处理规则。
- 核验训练集与 ZebraLogic 测试集的生成配置、实例去重和数据谱系，支持“无数据泄漏”的表述。
- 验证所有相关工作条目和 BibTeX。原始中文草稿含有若干 2025--2026 年条目和占位式题名，未核验前不得作为正式参考文献提交。
- 匿名投稿时，将 `Paper-1` 改为匿名自引格式，并移除“本系列第一篇/第二篇工作”等可识别表述。
- 以当年 AAAI 官方 CFP 和作者包为准重新确认页数、匿名、补充材料和 AI 使用披露要求。仓库中的 `aaai2026` 文档只可作为历史模板，不是当前规则的权威来源。

## 主张到证据的映射

| 主张 | 可在主文中使用的证据 | 不应超出的范围 |
|---|---|---|
| PRISM 在当前 ZebraLogic 设置中提高准确率 | 主结果表、每规模结果、方差和配对检验 | 不推广为所有 CSP 或所有模型均有效 |
| 修复轨迹记忆减少重复修复 | 修复轮数、停滞率、成对实例分析 | 不声称已证明真实错误原因 |
| 范式筛查有助于当前编码下的操作一致性 | 离线筛查、在线预检、消融 | 不称为自然语言语义认证或全域正确性证明 |
| 小规模范式可部分迁移 | L1 跨规模表 | 不称为稳定、普适的跨域泛化 |
| KnK 上存在可复用模式 | L2 的触发率、命中率与预注册比较 | 仅为探索性结果，不得作为通用结论 |
