## 摘要

约束满足问题（CSP）广泛存在于任务规划、调度和逻辑推理等场景。已有评测表明，大语言模型（LLM）在复杂 CSP 实例上存在显著的准确率骤降：当底层求解器冲突数超过一定阈值后，模型表现接近随机水平，且增加推理计算量收益有限。本文将这一失败归结为现有神经符号系统的两个结构性缺陷：其一，修复反馈粒度过粗，求解器仅返回笼统的不可满足信号，导致修复迭代陷入停滞；其二，问题实例之间相互独立求解，已积累的翻译经验与修复经验在任务结束后即被丢弃。

本文的核心主张是：**在推理步骤可被形式化验证的任务中，求解器可以作为经验准入的质量门控机制，从而将 LLM 记忆由非结构化的自然语言启发式转化为可机械筛选的结构化推理经验。** 基于该主张，本文提出 PRISM（Paradigm-guided Reasoning with Iterative Solver Memory）。离线阶段，PRISM 从求解轨迹中提炼两类结构化经验：从成功轨迹提炼携带 Z3 前后条件的正向范式，从失败轨迹挖掘携带 UNSAT 签名的错误范式；所有候选经验须通过一致性、效果和触发精度三重筛选方可入库。在线阶段，类型化修复轨迹记忆基于 UNSAT core 归因记录错误类型与修复动作，通过规范化 core 指纹的 Jaccard 相似度检测停滞、通过结构化动作匹配检测循环，并触发代价递增的四级策略切换，单题 LLM 调用具有 $R+2$ 的最坏预算上界。

在受控修复基准上（对完整形式化注入单条系统性错误约束，样本量 59–150/条件），从不相交小集挖掘的错误记忆将 direct_position 类错误的修复准确率从 32.7% 提升至 80.7%（3×3/3×4 规模，+48.0pp）、从 36.0% 提升至 67.0%（5×5/6×5 规模，+31.0pp），全部六个条件均通过 McNemar 配对检验（$p<0.05$ 至 $p<0.001$）（实测）。在 ZebraLogic 端到端评测上，【PLACEHOLDER：主表实验进行中，待补全各系统对比结果】。

## 1 引言

约束满足问题（Constraint Satisfaction Problem, CSP）要求在有限变量域上寻找同时满足全部约束的赋值，形式化为三元组 $\mathcal{P}=(\mathbf{X},\mathbf{D},\mathbf{C})$。以逻辑网格谜题为代表的自然语言 CSP 进一步要求系统从文本中识别变量与关系，并在多步推理中维护全局一致性。ZebraLogic 的系统评测表明，随着谜题规模增大与求解器冲突数上升，当前 LLM 的求解准确率急剧下降，Best-of-N 采样与自我验证等计算扩展手段收益有限【TODO：核验 Lin et al. 2025 引用与具体数字】。

神经符号方法将 LLM 的语言理解能力与 Z3 等形式化求解器结合，形成"翻译—执行—修复"流程。然而这类系统存在两个结构性缺陷。第一，**修复反馈粒度不足**：多数系统仅将原始报错字符串或笼统的 UNSAT 声明反馈给 LLM，模型无从知晓矛盾具体来自哪些约束，往往在连续多轮中生成近乎相同的错误修复，即修复停滞现象。第二，**跨题经验缺失**：每道题独立求解，成功的翻译模式、错误诊断与有效修复动作在任务结束后全部清零，相同类型的翻译错误在后续题目中被重复犯下。

智能体记忆研究（Reflexion、ExpeL、ReasoningBank 等）从另一维度探索了跨任务经验积累，但直接用于 CSP 推理时存在两个错配：其一，记忆单元为未经验证的自然语言文本，一条错误策略与一条正确策略在写入时无法机械区分；其二，任务级记忆粒度与 CSP 所需的推理操作级经验不匹配。

本文的关键观察是：CSP 求解具备通用智能体任务所不具备的性质——**每一个推理步骤都可以由 SMT 求解器机械检验**。若将候选推理步骤编码为 Z3 断言，求解器可以检查它是否与当前约束集一致、是否携带真实信息量（而非重言式）；当修复失败时，求解器返回的 UNSAT core（不可满足约束子集）为矛盾提供了精确的约束级定位。这使得经验在入库之前可以接受采样式的机械筛选，为记忆质量提供了自然语言记忆无法提供的门控信号。

本文据此提出 PRISM。围绕上述主张，本文的贡献如下：

1. **SMT 筛选范式库**：将正向范式（从成功轨迹提炼的推理操作）与错误范式（从失败轨迹挖掘的 UNSAT 签名规避模板）统一编码为带触发条件、操作模板与 Z3 前后条件的结构化经验，并以一致性、效果和触发精度三重筛选作为经验准入机制。
2. **类型化修复轨迹记忆**：基于 UNSAT core 归因进行约束级错误定位与错误类型分类，通过规范化 core 指纹的 Jaccard 停滞检测与结构化动作循环检测，驱动代价递增的四级策略切换，并给出单题 LLM 调用的最坏预算上界 $R+2$。
3. **系统性实证验证**：构建隔离修复能力的受控修复基准，并在复杂 CSP 端到端评测中与无记忆神经符号流程、自然语言经验记忆和直接检索式少样本基线对比，检验可机械筛选的结构化经验是否比未验证记忆更可靠。【TODO：端到端部分待主表实验完成后收紧措辞】

## 2 相关工作

**神经符号约束推理。** Logic-LM 建立了 LLM 翻译、求解器执行、LLM 修复的三阶段框架；SatLM 将问题形式化为 SAT 实例；后续工作（Logic-LM++、VERUS-LM、Logic.py、VERGE）改进了修复选择与冲突定位【TODO：核验各引用】。与 VERGE 的区别：VERGE 使用最小修正子集（MCS）在单题内定位冲突，计算代价较高；PRISM 使用 Z3 原生 UNSAT core 进行约束级归因，并在此之上建立跨题记忆与停滞检测——后两者不属于 VERGE 的系统设计。上述工作均将每道题视为独立实例，不维护跨题经验，也不建模修复是否陷入停滞。

**LLM 迭代自修复。** Self-Refine、Reflexion 等通过生成—反馈—修正循环提升输出质量，但自反馈信号在缺少外部监督时不稳定。本文以 UNSAT core 替代原始报错作为修复信号，并以类型化记录区分语法错误、过度约束、语义翻转等情形。

**智能体记忆与经验蒸馏。** ExpeL、AutoGuide、ReasoningBank 等表明经验蒸馏能改善跨任务决策，但其记忆单元多为自然语言文本，质量控制依赖 LLM 自评。PRISM 的差异在于：只有通过求解器三重筛选的经验才能入库，且记忆作用于推理操作级而非任务级。

**可复用推理规则学习。** 解释基学习（EBL）从成功轨迹泛化可证明正确的模式，但依赖手工领域理论；CDCL 与 SMT 引理生成从失败分支积累经验，但作用域限于单次求解调用。PRISM 位于该谱系的特定位置：范式从含噪 LLM 轨迹中归纳（无需领域理论），以采样式 SMT 筛选提供概率性而非形式化的质量证书，且跨问题实例复用。

## 3 问题定义与总体框架

（CSP 三元组、求解轨迹 $\mathcal{T}=\langle (S_t,a_{t+1},S_{t+1})\rangle$、状态 $S_t=(C_t,\delta_t)$ 的定义沿用现有稿件，此处从略；符号表置于正文，完整符号词汇表见附录 A。）

**求解范式** 定义为五元组 $P=(\mathcal{Q},\mathcal{O},\phi_{\text{pre}},\phi_{\text{post}},\mathcal{S})$：触发条件（约束类型集合与可选关系谓词 count_atleast/count_atmost/cooccur）、参数化操作模板、Z3 可形式化的前后条件断言、适用范围。范式与自由文本经验的本质区别在于它携带可被求解器机械检验的接口；筛选为采样式，提供概率性而非形式化的正确性证书。

**修复记录** 定义为六元组 $R_t=(e_t,U_t,\hat{U}_t,\alpha_t,o_t,U'_t)$：错误类型、UNSAT core、规范化 core 指纹（SHA-256）、结构化修复动作三元组（类型、目标约束、参数签名）及其句向量嵌入、修复结果、修复后新 core。错误类型取值于 {SYNTAX, OVER_CONSTRAINT, UNDER_CONSTRAINT, SEMANTIC_FLIP, SCOPE_ERROR, LEGACY}（与实现一致）。

**总体框架。** PRISM 将经验积累与单题推理解耦：离线阶段从训练轨迹提炼经求解器筛选的范式库 $\mathcal{L}$（正向）与 $\mathcal{E}$（错误）；在线阶段检索并注入范式引导推理，失败修复过程沉淀为修复轨迹记忆 $\mathcal{M}$，成功修复经批量重验证写回 $\mathcal{L}$。【TODO：Figure 1 总体框架图，建议用 academic-plotting 生成矢量图】

## 4 离线阶段：求解器门控的范式挖掘

**轨迹采集。** 对每道训练谜题以温度 0.7 运行 $r=3$ 次 LLM+Z3 求解（配置），每步记录 Z3 验证结果、域变化（`domain_sizes_before/after`）、约束类型签名与步骤类型。本文实际采集了 2,711 条 v2 轨迹（3×3/3×4 批次 1,200 条、4×5/5×5 批次 758 条、5×6/6×5 批次 453 条、GPT-4o-mini 混合批次 300 条）（实测）。

**关键决策点（KDP）识别。** 步骤 $(S_{t-1},a_t,S_t)$ 满足以下任一条件即为 KDP：
- **条件 A（域缩减）**：$\exists x\colon|\delta_{t-1}(x)|-|\delta_t(x)|\ge 2$（配置）；
- **条件 B（非平凡步骤类型）**：步骤被分类为链式传播（CHAIN）或矛盾消去（CONTRADICTION）；
- **条件 C（聚合信息增益）**：$G(t)=\sum_x \log_2(|\delta_{t-1}(x)|/|\delta_t(x)|)\ge \tau_{\text{ig}}=1.0$ 比特（配置）；
- **条件 D（成功修复）**：修复类步骤使求解器由 UNSAT 转为 SAT 且轨迹最终成功。

**聚类与 LLM 抽象。** KDP 表示为约束类型直方图、域大小分布、步骤类型独热与规模特征拼接的向量，L2 归一化后做完全连接余弦凝聚聚类，距离阈值 $\theta=0.25$（配置），支持度低于 $m_{\min}$ 的簇被丢弃【TODO：确定 $m_{\min}$ 最终取值——config 当前为 2（commit 8044f30 为适配现有轨迹量下调），符号表与代码默认值为 5，论文须与最终复跑配置一致并说明】。每个有效簇采样至多 10 个代表性 KDP，由 LLM 抽象为范式候选。

**三重筛选与 schema 不变量过滤。** 候选 $\hat P$ 须依次通过：
1. **一致性（soundness）**：在 $N_p=50$ 个满足触发条件的随机状态上实例化操作并检查 Z3-SAT，通过率须 $\ge \tau_{\text{snd}}=0.90$（配置）；
2. **schema 不变量过滤**：拒绝仅复述问题模式先验（如 $\text{And}(x\ge 1, x\le n)$ 类域边界）而不承载可复用推理的候选（实现于 `paradigm_verifier.is_schema_invariant`，实测该过滤器曾将早期"范式"全部识别为平凡项，是防止库污染的关键机制）；
3. **效果（effect）**：非矛盾性 $\text{Z3-SAT}(\phi_{\text{pre}}\wedge\text{op})$ 与非平凡性 $\text{Z3-SAT}(\phi_{\text{pre}}\wedge\neg\text{op})$ 须同时成立；
4. **触发精度（trigger precision）**：从 18 条标注约束池采样约束子集做集合交集匹配，要求非触发率 $\ge \tau_p=0.20$（配置；即触发器须在至少 20% 的随机样本上不触发，防止过度泛化）。

**错误范式挖掘。** 与正向分支互补，`ErrorParadigmExtractor` 从失败轨迹中按 UNSAT 签名聚合系统性翻译错误，构建携带规避指令与替换策略的错误范式库。当前 4×5/5×5 批次错误库含 88 条条目（实测）。【TODO：报告各批次错误库的最终规模、支持度分布与去重后类别】

## 5 在线阶段：范式引导推理与类型化修复记忆

**两层检索与一致性预检。** 第一层按约束类型签名与范式 scope 的 Jaccard 加权置信度打分，取 top-$K_{\text{top}}=3$、置信度下限 0.50（配置），并评估关系谓词以处理同类型不同数量的多重性；第二层由复杂度门控策略（约束数 $\ge 25$ 或已出现 UNSAT 时激活）调用 LLM 做语义判断（配置）。通过检索的范式在注入前须通过 Z3 一致性预检 $\text{Z3-SAT}(C_t\cup\{\text{op}(P)\})$，并以建议式提示注入，不作为硬约束。

**步验证与 UNSAT 归因。** LLM 每生成一步推断即由 Z3 验证。UNSAT 时执行两级归因：一级归因依据新断言 $a_t$ 是否属于 core $U_t$ 区分"当前推断引入矛盾"与"历史遗留错误（LEGACY）"；二级分类经 AST 解析（而非子串匹配）将前者细分为 SYNTAX / SCOPE_ERROR / SEMANTIC_FLIP / OVER_CONSTRAINT 等类型。

**停滞与循环检测。** Z3 的 UNSAT core 不唯一，直接比较会同时导致漏检与误检；PRISM 通过算子别名归一化与变量首现序重命名对 core 做规范化，指纹 $\hat U_t$ 取排序规范核的 SHA-256。停滞判据为最近 $k=3$ 条记录中**任意一对**规范核的 Jaccard 相似度 $\ge \tau_{\text{stag}}=0.75$（配置；取最大值对而非全对，以尽早预警、保留修复预算）。循环检测以结构化动作三元组精确匹配为主信号、句向量余弦 $\ge \tau_\ell=0.90$ 为回退信号（配置）：LLM 对同一动作的改写通常落在余弦 0.78–0.85 区间，结构信号可在语义阈值之下捕获真循环。

**四级策略切换与终止保证。** 检测触发后按代价递增升级：L1 切换修复目标（同 core 内换约束）→ L2 切换修复类型 → L3 从深度 $\le 5$ 的检查点栈回退（配置）→ L4 全量重新翻译（每题至多一次）。修复轮数上限 $R=5$（配置），最坏单题 LLM 调用为 $1+R+1=R+2=7$ 次；证明梗概见附录 B。

**经验写回。** 成功修复对应的候选范式先入暂存池（在线 $O(1)$），每 $K=50$ 道已解谜题触发一次批量去重与三重筛选复验（配置），通过者晋升入 $\mathcal{L}$；暂存池不参与在线检索，保证库内质量恒定。

## 9 误差分析：LLM 的系统性翻译失败模式

对失败轨迹的 UNSAT 签名聚合揭示了跨题目、跨属性反复出现的翻译失败模式（实测，来自错误范式挖掘）：**方向翻转**（将"A 在 B 左侧"的 $-1$ 偏移写为 $+1$）、**对称化错误**（将有向邻接扩展为双向 Or）、**间距误用**（以 $+2$ 偏移编码邻接）与**等式污染**（将位置比较写为相等）。其中类型化 directly_left 修复模板在 10 条轨迹中被独立触发验证（support=10，实测）。这些模式难以在系统设计阶段穷举，但可在运行时经 UNSAT core 指纹自动识别并建立规避模板——这构成错误范式作为独立贡献的证据基础。【TODO：用 error_*_v2 库的真实统计替换类别分布数字，绘制各规模错误类型直方图】

## 10 局限性

1. **概率性筛选**：三重筛选基于 $N_p=50$ 次采样，提供概率性而非形式化的正确性证书；至多 10% 的入库范式可能在未见状态上失效，在线一致性预检仅能过滤其中一部分。
2. **受控基准的外部效度**：受控修复基准通过扰动注入构造，其错误分布与自然翻译错误分布不完全一致；表 1 的增益应解读为"修复回路对系统性错误的恢复能力"，而非端到端提升的直接预测。
3. **正向范式的当前证据**：截至本稿，正向范式库由 schema 级条目主导，端到端增益尚未被证实（§7.2）；本文的主张强度据此收敛于错误范式与修复记忆。【TODO：若 4×5/5×6 批次挖掘出关系型范式并通过复评，此条相应改写】
4. **单域与单模型**：主要验证在逻辑网格类 CSP 与 GPT-4o-mini 上；跨 CSP 类型（AR-LSAT、KnK）与跨模型的范式可移植性为开放问题。
5. UNSAT core 规范化不能完全消除 core 非唯一性；结构不同但语义等价的核可能逃过停滞检测。

## 11 结论

本文提出 PRISM，其核心观点是：当推理步骤可被形式化验证时，求解器可以充当经验准入的质量门控，使 LLM 记忆从自然语言启发式转变为可机械筛选的结构化推理经验。PRISM 以三重筛选统一正向与错误范式的准入，以基于规范化 UNSAT core 的类型化修复记忆与四级策略切换约束修复过程。受控修复基准上的配对显著结果表明，从少量不相交数据中挖掘的错误记忆能大幅恢复系统性翻译错误（+31.0 至 +48.0pp）；端到端评测【PLACEHOLDER：待主表完成后总结】。未来工作包括将求解器门控记忆原理推广至定理证明与程序合成等可自动验证的推理场景。
