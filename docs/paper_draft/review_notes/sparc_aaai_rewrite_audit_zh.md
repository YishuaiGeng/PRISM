# SPARC AAAI 重写审查

## 论文主张

在任务给出可判定答案空间条件的 LLM 约束形式化中，SPARC 通过 SAT 后结构门控识别一部分可满足但错误（SBW）的输出，并支持结构性拒答；冻结 ZebraLogic 约束状态上的证据支持这一诊断结论，但不验证形式化语义正确性，也不证明候选补全的端到端修复收益。

## 术语账本

| 规范术语 | 首次定义 | 不再混用的表述 |
|---|---|---|
| SPARC | SAT 后结构门控及其探索性候选恢复扩展 | 已验证修复器、语义修复器 |
| SBW | 来自 SAT 状态、但任务答案错误的作答 | SAT 错误、错误 SAT 输出 |
| 答案投影 | 在答案变量集合 $V_A$ 上的模型投影 | 完整模型空间、答案空间不区分 |
| 结构门控 | 对任务结构谓词 $g_\pi$ 的求解器判定 | 唯一性检查、探针、门控随意切换 |
| 结构通过的答案 | 求解器确定满足结构条件的答案 | 已验证答案、正确答案 |
| 结构性拒答 | 求解器确定结构条件不成立后的非作答 | 其他非作答 |
| 未验证兼容输出 | UNKNOWN 或无法构造投影时的历史放行输出 | 门控通过 |
| 候选补全 | 由模型差异提示并经区分性验收的候选约束 | 语义修复、正确性修复 |
| 答案模式校验 | 历史 ZebraLogic 流水线中使用键名的前置校验 | 无金标端到端评测 |

## 主张—证据映射

| 主张 | 现有证据 | 状态 |
|---|---|---|
| 结构门控能识别一部分观测到的 SBW 输出 | 冻结、通过答案模式校验的 SAT 状态上，baseline 为 72/78，aggressive 为 69/73 | 支持，限 ZebraLogic 当前投影编码 |
| 结构门控带来覆盖代价 | 对应正确输出中，6/62 与 15/63 被判为非唯一 | 支持 |
| 补全预算改变风险—覆盖率操作点 | 同一完整轨迹的确定性前缀重放 | 支持，但仅为条件性重放 |
| SPARC 修复提升端到端表现 | 端到端臂只有 29.2%/33.2% 共享门控前约束 | 不支持，需要新实验 |
| 跨模型或跨任务泛化 | GPT-4o 单次运行、AR-LSAT 40 题实现探针、未配对组件运行 | 不支持，移出主文结论 |

## 本次重写

- 标题、摘要和引言统一定位为“SAT 后结构筛查与选择性作答”。
- 相关工作改为按技术机制组织，避免逐篇罗列。
- 删除主文中的错误类型学表和有限进展命题包装，保留其作为方法边界的简洁说明。
- 实验主线只保留冻结状态诊断与完整轨迹风险—覆盖率重放。
- 未配对端到端、组件、模型、AR-LSAT 和时延账本默认不编译进主文；将 `\includeexploratorytrue` 作为内部审计开关。
- 局限性改为直接说明结构条件、接口、oracle-schema 与证据外部效度的边界。

## 投稿前阻塞项

1. 用冻结的相同门控前约束执行前瞻性配对实验，才能将“候选补全/修复”保留为核心贡献。
2. 移除或证明答案模式校验可由题面独立恢复；否则固定状态结果必须持续标为 oracle-schema 条件下的诊断。
3. 显式指定答案变量白名单，并将 UNKNOWN/无法构造投影的输出从“通过门控”中分离，或改为 fail-closed。
4. 对选择性作答加入覆盖率匹配的随机拒答、置信度拒答或多形式化一致性基线；否则不能证明门控优于“少答一些”。
5. 正式英文稿必须迁移到 AAAI 模板并按当年 CFP 的主文页数、匿名与补充材料规则重新排版。现有 `aaai2026_prism/main_final.tex` 是另一篇 PRISM 稿件，不能直接覆盖。
