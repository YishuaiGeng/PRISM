Agent Memory与求解轨迹调研：相关工作、核心gap与方向可行性分析

Agent Memory 三条主线

现有工作从三个维度演进：情节记忆（存储历史轨迹）→ 语义记忆（抽象为规则/建议）→ 程序记忆（可执行技能/范式）。你想做的恰好在最前沿的第三层。

Reflexion 2023 NeurIPS基础层

Shinn et al. — 语言强化学习：用自然语言 self-reflection 作为 "verbal gradient"

失败后让 LLM 反思"哪里出错、下次怎么做"，把反思写入 episodic memory 供下次参考。在 AlfWorld/HotpotQA 上显著提升。
局限：单任务内记忆，不跨任务积累；反思是自由文本，无结构，难以检索；在长期任务中记忆衰减。

ExpeL 2024 AAAI语义抽象层

Zhao et al. — 跨任务 insight 提取：比较成功/失败轨迹对，抽取"经验规则"

三阶段：(1) 用 Reflexion 收集成功+失败经验池；(2) 让 LLM 从轨迹对中提取 insight（ADD/UPVOTE/DOWNVOTE/EDIT 四种操作维护 insight 列表）；(3) 测试时把 insights 拼入 prompt。首次实现跨任务经验转移。
局限：insights 是平铺列表，随积累线性增长，tokens 爆炸；所有 insights 一刀切注入，与当前问题不相关的也进来；在 CSP/推理域未验证。

ReasoningBank Sep 2025最新

Ouyang et al. — 从成功+失败经验蒸馏高层推理策略，结合 test-time scaling

不存原始轨迹，而是提取"结构化知识单元"（高层策略 + 防错建议），同时学习成功经验和失败教训。引入 MaTTS（memory-aware test-time scaling）：更多计算 → 更丰富对比信号 → 更高质量 memory。在 web browsing/SWE 上 outperform 存储原始轨迹的方法。
局限：仍是通用任务域（网页浏览、代码），记忆单元为自然语言，没有形式化结构；不能利用 solver 的精确反馈。

ProcMEM / MACLA 2025-2026程序记忆层

— 从轨迹抽取可执行技能（Skills），替代情节式参考

MACLA：将 2851 条 ALFWorld 轨迹压缩为 187 个可复用程序（15:1 压缩比），通过语义抽象和去重实现。ProcMEM：用非参数化 PPO 门控验证技能质量，维持高质量紧凑的程序记忆库。
局限：技能是自然语言 + 代码混合，验证靠 LLM 自评；没有形式化验证机制；应用在具身/GUI 任务，不是推理/CSP 域。

AgentHER / ERL 2025-2026失败轨迹利用

失败经验 hindsight 重标注 + heuristic 提取

ERL 发现：失败经验提取的 heuristics 在"搜索类"任务上（+14.3%）优于成功经验，因为它提供负约束（"不要走这条路"）。成功经验在"执行类"任务上更好（+9.0%）。原始轨迹直接拼入不仅无效，反而 -1.9%。这支持了"轨迹需要抽象为 heuristics 才有价值"的结论。

ZebraLogic + GridPuzzle 错误分析 2024-2025谜题域基础

Lin et al. / Tyagi et al. — LLM 在 CSP/谜题上的错误模式分析

ZebraLogic 发现：精度随 Z3 冲突数增加急剧下降；backtracking-based approach 显著提升。GridPuzzle 分析：最主要错误是"wrong reasoning"和"elimination 错误"。关键发现：LLM 在需要反复回溯的 CSP 上表现极差，但这正是"求解范式"最有价值的地方——如果有正确的约束传播策略，可以减少需要回溯的次数。

关键 Gap — 你的想法恰好对准了三个未被填补的空白

Gap 1：没有领域感知（domain-aware）的记忆结构

ExpeL/ReasoningBank 的记忆单元是自由文本 heuristics，适合通用任务。但 CSP 求解有结构性知识——约束传播规则、消去策略、回溯条件——这些有明确的形式语义，可以被精确验证。没有工作为 CSP 域设计利用这一结构的记忆表示。你的求解轨迹每一步都能用 solver 验证，这是通用 Agent 任务完全不具备的优势。

Gap 2：修复轨迹从未被建模为学习信号

现有神经符号 Agent 工作（Logic-LM 系列）中，每次修复迭代是独立的，不记录"这次修的是什么错误、怎么改的、改成功了没有"。这正是 stagnation 的根本原因。ERL 证明失败轨迹很有价值，但它针对的是任务执行轨迹，不是形式化修复轨迹（修复轨迹有更精确的结构：错误类型 → 修复动作 → solver 验证结果）。

Gap 3：求解"范式"从未被形式化提取

人类解谜题有范式：先用单约束确定确定的值，再通过消去缩小域，最后在必要时回溯。这些策略在传统 CP 文献中早已被研究（arc consistency, forward checking 等），但从未被用来指导 LLM 生成求解轨迹，也没有工作研究"LLM 能否从过去的成功轨迹中提炼出这类策略，并在新谜题上复用"。这是你的核心创新空间。

一句话定位你的工作

把 ExpeL 的"经验抽象"思想 + Reflexion 的"修复记忆"思想，移植到神经符号 CSP 求解域，并利用 solver 的形式化验证能力，将记忆单元从自由文本 heuristics 升级为可验证的求解范式（solver-verified solving paradigms）。

方向设计：两层记忆 + 范式蒸馏

模块 1

修复轨迹记忆（Repair Trajectory Memory）

解决 stagnation 问题，记录"尝试了什么、为什么失败"

核心数据结构

每次修复迭代生成一条 repair record：

错误类型
over/under/syntax/flip...
→

→

三个用途

  • 停滞检测：若连续 K 条记录的 UNSAT core 集合相同，触发策略切换
  • 循环检测：若修复动作与之前某条记录高度相似，阻断重复并注入"已尝试 X，无效，请尝试 Y"
  • 回溯引导：若所有局部修复失败，提示 LLM 回退到上一个确定正确的约束集合重新来过

对比 Reflexion

Reflexion：自然语言反思，不可验证
你的：每条记录有 solver 验证标签，精确

实现难度

中低。数据结构简单，主要工程在 UNSAT core 解析 + 相似度检测

模块 2

求解范式蒸馏（Solving Paradigm Distillation）

从成功求解轨迹中提炼可复用的推理策略，跨谜题实例迁移

核心思想

人类解谜题有固定策略：① 单约束必然推断（naked single）→ ② 消去法缩小域 → ③ 候选集唯一则确认 → ④ 必要时试探+回溯。如果 LLM 能从成功轨迹中提炼类似的策略模板，就能在新谜题上直接用策略引导，而非从头摸索。

蒸馏流程

范式是否产生有效推断

范式形式（初步设想）

  • 触发条件：当某类约束组合出现时（如"相邻 + 排除"约束共存）
  • 推断步骤：应用何种消去或传播操作
  • 验证条件：推断结果可用 Z3 验证是否与现有约束相容
  • 适用范围标签：该范式在哪类谜题规模/结构下有效

对比 ExpeL

ExpeL：自由文本 insights，平铺存储
你的：结构化范式 + solver 可验证条件

最大创新点

范式有形式语义，可以检查"范式建议的步骤是否与当前约束集 consistent"

模块 3

跨问题类型泛化实验

验证范式是否能从 Zebra 迁移到其他 CSP/逻辑推理场景

泛化层次

  • Level 1（同域泛化）：同类型谜题，不同规模/难度。验证范式在 3×5 谜题上学到的策略能否用于 5×6 谜题
  • Level 2（结构迁移）：从 Zebra 到其他逻辑网格谜题（数独、Hitori、NQueens 的推理结构），看共享策略
  • Level 3（语义迁移）：从谜题到真实约束推理（调度、资源分配），看"约束传播范式"能否跨语义域

评估指标

  • 使用范式 vs 不使用范式的求解成功率对比
  • 平均 LLM 调用次数（效率）
  • 修复迭代轮数（收敛速度）
  • 范式检索准确率（是否检索到了有效范式）

可行性评估

✓ 核心判断：在 3 个月内，模块 1 + 模块 2（限 Level 1）是高可行性的一篇完整论文

有利条件

① 你有现成基础设施：第一篇的神经符号求解框架 + 第二篇的谜题生成器，直接用于生成训练轨迹和评估数据集，零冷启动成本。

② Z3 提供精确的记忆验证机制：每条修复轨迹记录的 solver 结果是客观标签，不像 ExpeL 那样需要 LLM 自判"这次成功了没有"。

③ Zebra 谜题是范式提炼的理想场景：约束类型有限（相邻/排除/绑定/顺序），解空间结构清晰，比代码或网页任务更容易提炼出可验证的范式模板。

④ 差异化充分：ExpeL/ReasoningBank/ProcMEM 都在通用任务域，没有一篇工作在 CSP/形式推理域做记忆驱动的范式提炼，竞争稀少。

⚠ 主要风险：模块 2 的范式泛化（Level 2-3）可能在 3 个月内难以充分验证，建议 Level 1 为主，Level 2 为辅

风险点及应对

风险 1：范式提炼质量不稳定——LLM 从轨迹中提炼的范式可能过于具体（记住了特定谜题）或过于抽象（没有操作性）。
应对：用 solver 验证+成功率打分筛选范式；设计对比实验（细粒度 vs 粗粒度范式）。

风险 2：记忆检索效率——范式库增大后，如何高效检索与当前谜题相关的范式。
应对：初期用简单的约束类型特征匹配，不依赖语义检索，降低实现复杂度。

风险 3：Level 3 泛化可能结果不好——跨语义域迁移风险高。
应对：这部分作为 discussion/future work，不要作为主要贡献。

建议时间线（12 周）

第 1-2 周 设计修复轨迹数据结构，实现 stagnation + 循环检测；在前两篇基础上跑基线实验
第 3-5 周 生成大量 Zebra 求解轨迹（用第二篇生成器），实现基础范式提炼 pipeline
第 6-8 周 迭代优化范式质量，完成 Level 1 同域泛化实验，消融分析
第 9-10 周 Level 2 跨谜题迁移实验（数独/Knights-Knaves 等）
第 11-12 周 写作，整理消融，补实验

投稿目标建议

主线（模块 1 + 2 Level 1）完成度足够投 ACL/EMNLP 2026（找 2025 年底截稿）或 IJCAI 2026；如果 Level 2 泛化结果好，也可以冲 AAAI/NeurIPS。
