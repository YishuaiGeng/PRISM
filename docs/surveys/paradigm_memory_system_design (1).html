求解范式记忆系统详细设计：系统架构、范式表示示例、论文贡献框架

整体系统流程

虚线 = 经验反馈回路（成功轨迹持续补充范式库）；实线 = 主要执行流程；UNSAT 时触发修复记忆介入。

三个核心范式的具体表示（点击步骤切换）

最高频率范式，约 60% 的谜题步骤可用。对应人类解数独的 "naked single" 策略。

P1直接赋值范式（Direct Assignment）

触发条件某个自然语言约束形如"X 是/在 V"（直接绑定），或经过前序消去后变量域缩为单一值
推断操作令 domain(X) := {V}，然后对同一属性类的所有其他变量执行 domain(Y) := domain(Y) \ {V}
Z3 验证SAT(C_current ∧ X=V) 必须成立；同时 ∀Y≠X: SAT(C_current ∧ Y≠V)
适用范围任意规模谜题，触发频率高；可以串联多次直至无新赋值可得

具体例子（Zebra 3×3 谜题片段）

原始约束（自然语言）： "挪威人住在第1号房子里" "绿色房子紧接在白色房子左边" "养马的人住在蓝色房子旁边"
P1 触发：检测到直接约束 "挪威人→房子1" 推断操作： domain(nationality[house=1]) := {Norwegian} domain(nationality[house=2]) := {German, English, Swedish, Danish} domain(nationality[house=3]) := {German, English, Swedish, Danish} Z3 验证：SAT Norwegian ∧ house1 与现有约束一致
P1 串联触发（域缩减后）： domain(drink[house=2]) = {Milk}（由其他约束推导） 再次应用 P1： domain(drink[house=2]) := {Milk} domain(drink[house=1,3]) := domain \ {Milk} 本步骤通过 P1 两次应用，无需 LLM 重新推理 节省调用：省略 2 次 LLM token 消耗 Z3 验证：SAT 两次赋值均与约束一致

P2 是 Zebra 谜题中最"智能"的范式——它能将两个单独看起来不完整的约束组合起来，得出第三个约束。对应约束传播中的"弧一致性"推断。

P2链式传播范式（Chain Propagation）

触发条件两个约束共享同一变量，且可通过传递性导出新绑定。典型形式：pos(A)=k 已知 + pos(B)=pos(A)+1 → 可推出 pos(B)=k+1
推断操作沿约束链向前传播，每次用已知值替换约束中的变量，直到链条断开或得到确定赋值
Z3 验证验证推导出的赋值 SAT(C_current ∧ derived_assignment)；若 UNSAT 说明链条中有翻译错误
适用范围包含"左边/右边/紧挨"类相对位置约束的谜题，触发频率中等
例子： 约束1（已知）： pos(Norwegian) = 1 约束2（原始）： pos(BlueHouse) = pos(Norwegian) + 1 P2 触发：检测到"位置已知 + 相对约束"模式 传播步骤： pos(BlueHouse) = 1 + 1 = 2 Z3 验证：SAT 注意：若翻译错误把约束2写成 pos(B)=pos(A)-1 则 pos(BlueHouse)=0（超界），Z3返回：UNSAT → 触发修复记忆，标记"相对位置方向可能翻转"
P3 最难提炼，因为它需要 LLM 做"假设—验证—否定"三步推理。但也是最有价值的：它解决了复杂谜题必须回溯的问题。

P3矛盾消去范式（Contradiction Elimination）

触发条件当前变量域 |domain(X)|≥2 且 P1/P2 无法进一步推断；选择一个候选值 v 做试探赋值
推断操作令 X=v（试探），传播后调用 Z3；若 UNSAT 则 domain(X) := domain(X) \ {v}；若域缩为1则触发 P1
Z3 验证UNSAT core 必须包含试探约束 X=v，否则说明是其他约束翻译有误（区分两种 UNSAT 来源）
适用范围当 P1/P2 穷尽仍无法确定时激活；UNSAT core 检查提供了翻译错误诊断的副产品
例子：domain(color[house=3]) = {Green, White} P3 触发：试探 color[house=3] = Green 传播所有相关约束... Z3 结果：UNSAT UNSAT core 包含： { X=Green } ∪ { "绿色在白色左边" } → core 包含试探约束 ✓（是试探导致的，不是翻译错） → 消除：domain(color[house=3]) := {White} → P1 触发：color[house=3] = White 若 core 不含试探约束： → 是其他约束的翻译错误 → 修复记忆记录此诊断

修复轨迹记忆的数据结构与机制

单条修复记录（Repair Record）

{ "timestamp": 3, // 第几次迭代 "error_type": "over_constraint", "unsat_core": ["C3", "C7"], // 哪两条约束矛盾 "core_fingerprint": "hash(sorted(['C3','C7']))", "repair_action": { "type": "relax", "target_constraint": "C7", "summary": "将'紧挨'改为'同侧'" }, "outcome": "UNSAT", // 修复后仍失败 "new_core": ["C3", "C7", "C9"] // 修复后的新 core }

停滞检测算法

连续 K 条记录的 core_fingerprint 集合 Jaccard 相似度 ≥ 0.8 → 触发停滞警报。系统注入提示："以下修复路径已被尝试过，请换一个方向" + 历史记录摘要。

循环检测算法

若当前 repair_action.summary 的嵌入向量与历史中某条记录的余弦相似度 > 0.92 → 阻断重复，注入："此修复在第 N 轮已尝试，结果为 UNSAT，请尝试修改不同约束"。

策略切换触发条件

当停滞或循环检测触发时，按优先级依次尝试：

1切换目标约束——从修 C7 换成修 UNSAT core 里的另一条约束 2切换修复类型——从"放松约束"换成"重新翻译约束" 3回退检查点——回到上一个 SAT 状态，从另一方向重新推导 4降级回溯——放弃当前形式化，重新从自然语言翻译（最后手段）

经验写回机制

当最终求解成功时，回溯修复历史，提取"有效修复"（最终导致 SAT 的那条记录），将其模式化后加入范式库作为候选。下一次遇到相同 core_fingerprint 时，直接建议已知有效的修复动作。

论文框架 + 贡献表述

建议标题

Learning Solver-Verified Solving Paradigms from CSP Trajectories for LLM-Based Neuro-Symbolic Reasoning

或更聚焦的版本：From Experience to Paradigm: Memory-Augmented Neuro-Symbolic Agents for Constraint Satisfaction

核心论点（一句话）

LLM 在 CSP 上重复失败的根本原因是"每次从零推理，不积累求解经验"；通过从成功轨迹中提炼形式可验证的求解范式，并以修复记忆阻止重复失败，可以同时解决"不会"和"学不快"两个问题。

四项核心贡献

  1. 范式表示语言（Paradigm Representation）

    定义 CSP 求解范式的结构化表示：触发条件（约束类型模式）+ 推断操作 + Z3 可验证前/后条件 + 适用范围标签。这是与 ExpeL/ReasoningBank 的根本区别——记忆单元有形式语义。

  2. 范式提炼 pipeline（Paradigm Distillation）

    从 N 条成功轨迹 → 关键决策点标注 → 跨轨迹聚类 → LLM 抽象 → Z3 验证筛选。量化压缩比（N 条轨迹 → K 个范式，K≪N）和范式覆盖率。

  3. 修复轨迹记忆（Repair Memory）

    带停滞检测和循环阻断的有结构修复记忆系统，配合四级策略切换。直接解决 Logic-LM 系列工作的 stagnation 问题，有精确的量化指标（平均修复轮数下降 X%）。

  4. 泛化实验（Generalization Study）

    Level 1：同域不同规模；Level 2：跨谜题类型（数独、Knights-Knaves 等）。建立"范式迁移率"评估指标——一个范式在多少比例的新谜题上成功触发并有效。

最有力的 Ablation

去掉范式 → 去掉修复记忆 → 去掉 Z3 验证（只用 LLM 自评）→ 去掉策略切换。每一步都应该有明显的性能下降，证明各模块独立有效。特别是"Z3 验证 vs LLM 自评"这个对比，直接回应 ProcMEM 等工作的局限。
