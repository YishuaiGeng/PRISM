Step 1 · 初始化项目结构

请帮我初始化一个名为 prism 的 Python 研究项目。 项目结构如下： - 根目录：prism/ - 子目录：prism/core/, prism/offline/, prism/paradigm_library/, prism/online/, prism/evaluation/, scripts/, data/, tests/, config/, paradigm_store/ - 在每个 Python 包目录创建 __init__.py - 创建 requirements.txt，包含：z3-solver, sentence-transformers, scikit-learn, anthropic, openai, wandb, pydantic, pytest, pytest-cov, pyyaml, numpy, pandas - 创建 setup.py，package name 为 prism，python_requires=">=3.10" - 创建 config/default.yaml，包含 llm_backend(anthropic/openai), model_name, thresholds(stagnation_jaccard:0.75, loop_cosine:0.90, paradigm_soundness:0.90, min_support:5), paths(data_dir, paradigm_store, trajectory_dir) - 创建 README.md，标题为 "PRISM: Paradigm-guided Reasoning with Iterative Solver Memory"

Step 2 · 生成核心数据类和接口

请在 prism/paradigm_library/schema.py 中创建以下 Pydantic dataclass： 1. ErrorType(Enum): SYNTAX, OVER_CONSTRAINT, UNDER_CONSTRAINT, SEMANTIC_FLIP, SCOPE_ERROR, LEGACY 2. Outcome(Enum): SAT, UNSAT 3. RepairAction: fields = type(str), target_constraint(str), summary(str), embedding(Optional[List[float]]) 4. RepairRecord: fields = iteration(int), error_type(ErrorType), unsat_core(List[str]), core_fingerprint(str), repair_action(RepairAction), outcome(Outcome), new_core(Optional[List[str]]) 5. Paradigm: fields = id(str), name(str), trigger(dict), operation(str), pre_condition(str), post_condition(str), scope(List[str]), confidence(float), support_count(int), source_cluster(int), created_at(datetime) 每个类都加 docstring 说明用途。在 __init__.py 中统一导出。

Step 3 · 创建 Z3 Solver 接口

请在 prism/core/solver.py 中创建 Z3SolverWrapper 类，要求： 1. __init__(self): 初始化 z3.Solver() 2. add_constraint(self, constraint_str: str) -> bool: 将字符串形式的 Z3 约束添加到 solver，返回是否语法合法 3. check(self) -> str: 返回 "SAT" / "UNSAT" / "UNKNOWN" 4. get_unsat_core(self) -> List[str]: 当结果为 UNSAT 时，返回最小不满足约束子集（使用 z3 unsat_core 接口）；需要先用 assert_and_track 追踪约束 5. get_model(self) -> dict: 当 SAT 时返回变量赋值字典 6. reset(self): 清空所有约束 7. clone(self) -> Z3SolverWrapper: 复制当前 solver 状态用于试探性验证 8. verify_paradigm_soundness(self, current_constraints: List[str], paradigm_assertion: str, n_trials: int = 20) -> float: 在 n_trials 个随机约束子集上测试范式的 soundness，返回通过率 加完整的类型注解和 docstring，import z3，异常用自定义 SolverError。

Step 4 · 创建修复记忆模块

请在 prism/online/repair_memory.py 中创建 RepairMemory 类： 1. __init__(self, config: dict): 读取 stagnation_jaccard 和 loop_cosine 阈值 2. append(self, record: RepairRecord) -> None: 追加记录，自动计算 core_fingerprint（sorted unsat_core 的 hash），如果 repair_action 有 summary，用 sentence-transformers 计算 embedding 并存入 3. detect_stagnation(self, k: int = 3) -> bool: 取最近 k 条记录，计算 core_fingerprint 集合的 Jaccard 相似度，超过阈值返回 True 4. detect_loop(self, action: RepairAction) -> bool: 计算新 action 的 embedding 与历史所有 embedding 的余弦相似度最大值，超阈值返回 True 5. get_history_summary(self) -> str: 生成给 LLM 的历史提示，格式："已尝试修复 N 次，包括：[第1次: 修改C3，结果UNSAT; 第2次: ...]，请尝试不同策略" 6. get_successful_repairs(self) -> List[RepairRecord]: 返回 outcome==SAT 的记录 7. clear(self) -> None: 重置（每道新谜题前调用） 8. to_dict(self) -> dict / from_dict(cls, d: dict): 序列化支持 用 lazy import sentence-transformers（避免无 GPU 时慢初始化）。

Step 5 · 创建策略切换状态机

请在 prism/online/strategy_switcher.py 中创建 StrategySwitcher 类： 策略级别枚举 SwitchLevel(Enum): L1_SWITCH_TARGET, L2_SWITCH_TYPE, L3_REVERT_CHECKPOINT, L4_FULL_RETRANSLATE 类方法： 1. __init__(self, memory: RepairMemory) 2. should_switch(self) -> Optional[SwitchLevel]: 检查 memory 的停滞/循环状态，返回应触发的级别（None 表示继续正常修复） 3. get_switch_prompt(self, level: SwitchLevel, current_state: dict) -> str: 根据级别生成注入给 LLM 的策略切换提示文本 - L1: "请换一个约束作为修复目标，已尝试修改 {已修改约束}，请尝试 {UNSAT core 中其他约束}" - L2: "请换一种修复方式，已尝试 {已用修复类型}，请尝试 {候选修复类型}" - L3: "回退到检查点 {checkpoint}，从另一个推断方向重新开始" - L4: "重新从自然语言翻译所有约束，忽略之前的形式化" 4. save_checkpoint(self, state: dict) -> None: 保存当前 SAT 状态作为回退点 5. get_checkpoint(self) -> Optional[dict]: 获取最近检查点 加完整注解和单元测试用例注释。

Step 6 · 创建范式库

请在 prism/paradigm_library/library.py 中创建 ParadigmLibrary 类，使用 SQLite 持久化： 1. __init__(self, db_path: str, solver: Z3SolverWrapper): 初始化 SQLite 连接，建表（若不存在） 2. add(self, paradigm: Paradigm, verify: bool = True) -> bool: 添加范式；若 verify=True，先调用 Z3SolverWrapper.verify_paradigm_soundness，低于阈值拒绝添加 3. retrieve(self, constraint_types: List[str], top_k: int = 3) -> List[Paradigm]: 基于约束类型特征检索最匹配的 top_k 个范式，按 confidence 排序 4. update_confidence(self, paradigm_id: str, new_score: float) -> None: 更新验证置信度 5. get_all(self) -> List[Paradigm]: 返回全部范式 6. delete(self, paradigm_id: str) -> None 7. save_json(self, path: str) / load_json(self, path: str): JSON 导入导出 8. stats(self) -> dict: 返回 {total, avg_confidence, avg_support, scope_distribution} 建表 SQL：paradigms(id TEXT PRIMARY KEY, name, trigger_json, operation, pre_condition, post_condition, scope_json, confidence REAL, support_count INT, source_cluster INT, created_at TEXT)

Step 7 · 创建实验评估指标

请在 prism/evaluation/metrics.py 中实现以下评估函数： 1. solve_accuracy(results: List[dict]) -> float: 正确求解率（ground truth 匹配） 2. avg_llm_calls(results: List[dict]) -> float: 平均 LLM 调用次数 3. avg_repair_rounds(results: List[dict]) -> float: 平均修复迭代轮数 4. paradigm_trigger_rate(results: List[dict]) -> float: 有范式被触发的步骤占总步骤比例 5. paradigm_hit_rate(results: List[dict]) -> float: 触发范式后推断正确的比例 6. paradigm_transfer_rate(source_results: List[dict], target_results: List[dict]) -> dict: 计算 L1/L2 迁移率，返回 {trigger_rate, hit_rate, delta_accuracy} 7. stagnation_reduction(baseline_results: List[dict], prism_results: List[dict]) -> float: 修复停滞率下降比例 8. generate_report(results: List[dict], library_stats: dict) -> str: 生成 markdown 格式的实验报告 每个函数都有详细 docstring 说明 results dict 的预期格式。
