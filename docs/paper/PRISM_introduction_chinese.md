## 1 引言

约束满足问题（Constraint Satisfaction Problems，CSP）是一类要求智能体在离散变量域上同时满足多个相互关联约束的推理任务，在任务规划、资源分配与调度优化等现实应用中具有广泛意义（Dechter, 2003）。近年来，以大型语言模型（LLM）为核心的推理系统在自然语言理解和数学推理等领域展示出令人瞩目的能力，然而在结构化约束推理方面却面临根本性挑战。ZebraLogic（Lin et al., ICML 2025）的系统性评测揭示了一个"复杂度诅咒"现象：随着谜题规模增大，所有被测模型的精度均出现急剧下降，当 Z3 求解器的冲突数超过 20 时甚至接近随机水平，且增加推理计算量（Best-of-N 采样、自我验证提示）的收益极为有限。这一现象表明，当前 LLM 在 CSP 求解上的失败并非单纯的规模效应，而是根植于对复杂约束结构缺乏系统性求解策略这一深层原因。

**神经符号方法**通过将 LLM 与外部形式化求解器（如 Z3、Prover9）相结合，为缓解上述问题提供了一条有前景的路径（Pan et al., 2023; Olausson et al., 2023; Ye et al., 2023）。此类方法通常采用"翻译-执行-修复"三阶段流水线：LLM 负责将自然语言约束翻译为符号表示，求解器执行形式化推理，当执行失败时再由 LLM 根据错误信息尝试修复翻译。尽管这一范式相较于纯语言模型推理取得了显著改善，但在实际应用中暴露出两个根本性缺陷。其一，**修复机制的低效性**：现有工作（Pan et al., 2023; Kirtania et al., 2024）明确观察到修复迭代数轮后精度趋于平台（stagnation）的现象——其根本原因在于反馈信号（原始报错字符串或通用不满足性声明）不携带足够的语义信息，LLM 被迫在无新信息的情况下反复生成近似相同的错误修复；当求解器返回 UNSAT 时，系统既无法区分"谜题本身无解"与"翻译引入了额外约束"，也无法识别当前修复是否陷入循环。其二，**经验积累的缺失**：每道谜题独立求解，累积的翻译经验、错误诊断与修复轨迹在任务切换后全部清零——相同类型的约束翻译错误在不同谜题中将被重复犯下。

**智能体记忆方法**从另一维度探索了跨任务经验积累问题（Shinn et al., 2023; Zhao et al., 2024; Ouyang et al., 2025）。通过对比成功与失败轨迹、提炼可复用的启发式规则或结构化知识单元，此类方法在网页导航、具身任务和软件工程等领域展现出有效的跨任务迁移能力。然而，这一技术路线在应用于 CSP 推理时面临两个固有局限：一方面，现有工作的记忆单元均为自然语言散文，没有任何形式化验证机制——一条错误的策略与正确策略在存储时完全无法区分，质量保证完全依赖 LLM 自评（Huang et al., 2024; ERL, 2026）；另一方面，现有记忆框架以任务为记忆粒度，而 CSP 求解所需的是推理操作级别的精细经验（"当约束类型 X 与 Y 共存时，应用传播操作 Z"），两者存在根本性的粒度不匹配。

上述两类工作的局限促使我们寻找将二者互补优势融合的可能性。我们注意到，CSP 求解领域存在一个通用智能体任务所不具备的关键性质：**每一个推理步骤均可被 Z3 等 SMT 求解器形式化验证**。这一性质从根本上改变了经验积累的条件：范式在写入记忆库之前可以接受机械化的正确性认证，而非依赖 LLM 的主观判断；修复失败不再是模糊的"任务未完成"信号，而是携带着精确约束定位信息的 UNSAT core——即引发矛盾的最小约束子集。CSP 领域的可验证性将经验积累从噪声自监督信号升级为形式化接地（formally grounded）的学习过程，这正是本文工作的核心立足点。

基于上述洞察，本文提出 **PRISM**（**P**aradigm-guided **R**easoning with **I**terative **S**olver **M**emory），一个面向 CSP 求解的记忆增强神经符号智能体框架。PRISM 以两阶段架构运作：在**离线阶段**（一次性执行），系统在可控难度谜题生成器（基于前序工作）产出的训练谜题上收集成功求解轨迹，识别其中的关键决策点（Key Decision Points，KDP），通过跨轨迹聚类与 LLM 抽象提炼结构化求解范式候选，最终经 Z3 三重验证（soundness、effect、trigger precision）筛选后写入**范式库**（Paradigm Library）。在**在线阶段**（每道谜题执行），系统首先根据当前约束类型特征从范式库检索相关范式并注入提示，引导 LLM 做出更高质量的推理步骤；Z3 对每步推断进行实时验证，当返回 UNSAT 时，系统提取 UNSAT core 进行归因判断，并将类型化的修复记录（error type、UNSAT core、repair action、outcome）写入**修复轨迹记忆**（Repair Trajectory Memory）。修复记忆通过 Jaccard 相似度检测停滞（连续多轮 UNSAT core 无实质变化）、通过余弦相似度检测循环（修复动作与历史高度重复），一旦检测到上述情形即触发四级策略切换协议（L1：切换修复目标约束；L2：切换修复类型；L3：回退至最近 SAT 检查点；L4：全量重新翻译），从机制层面打破修复停滞。成功求解后，有效修复经验通过写回机制（write-back）补充范式库，实现推理时经验的持续积累。

我们在 ZebraLogic 基准（Lin et al., 2025）的六个规模层级（3×5 至 6×6）上对 PRISM 与七条基线进行了系统评估，基线涵盖纯链式思维推理、本系列前序工作（LLM+Z3，无记忆）、Logic-LM、ExpeL（CSP 适配版）、经过 Z3 验证的直接少样本注入（Verified Few-Shot），以及 PRISM 的两个消融变体。实验结果表明，PRISM 在所有规模上均显著优于对比基线，其中在最困难的 6×6 谜题上相较于前序工作提升幅度最大，佐证了"经验积累对困难问题的价值随复杂度增加而增大"这一核心论点。进一步分析表明，范式库在同域不同规模间具有较好的迁移能力，修复轨迹记忆将停滞触发率显著降低，每道谜题平均修复轮数和 LLM 调用次数均有明显减少。

本文的主要贡献如下：

1. **求解范式表示**（Solving Paradigm Representation）：提出首个面向 CSP 推理的形式可验证求解范式表示，以结构化四元组（触发条件、推断操作、Z3 前条件、Z3 后条件）替代现有记忆工作中的自由文本启发式，实现了记忆单元质量的机械化认证。

2. **范式提炼流水线**（Paradigm Distillation Pipeline）：设计全自动的离线范式提炼流水线，从原始求解轨迹出发，经 KDP 识别、跨轨迹聚类、LLM 抽象和 Z3 三重验证，将约 1500 条训练轨迹压缩为约 35 个高置信度范式（~40:1 压缩比），无需人工标注。

3. **类型化修复轨迹记忆**（Typed Repair Trajectory Memory）：提出首个将约束修复过程建模为可学习结构化轨迹的记忆系统，包含基于 UNSAT core 的 Jaccard 停滞检测、基于动作嵌入的余弦循环检测与四级策略切换协议，从机制层面解决了现有神经符号系统普遍面临的修复停滞问题。

4. **范式迁移率评估体系**（Paradigm Transferability Study）：建立"范式迁移率"这一新的评估维度，系统量化范式在同域不同规模（L1）和跨谜题类型（L2）场景下的触发率与命中率，为神经符号经验积累的泛化能力研究提供方法论基础。
