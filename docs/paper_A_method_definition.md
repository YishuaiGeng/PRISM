## 一、你目前的方法到底是什么（从草稿解剖）

草稿的 PRISM 可分解为一条流水线加两个记忆插件：

```
基础流水线（继承 Logic-LM 谱系，非贡献）：
  NL 问题 → LLM 翻译为 Z3 约束 → 求解 → UNSAT 时以 UNSAT core 为信号迭代修复

插件一（跨题，成功侧）：SMT 筛选范式库
  成功轨迹 → KDP 识别 → 聚类 → LLM 抽象 → Z3 三重筛选 → 检索注入

插件二（题内 + 跨题，失败侧）：类型化修复轨迹记忆
  UNSAT core 归因 → 错误分型 → 停滞/循环检测 → L1–L4 策略升级 → 经验写回
```

**隐含的三个前提假设（实证均不成立）**：

| 假设 | 实证结果 |
|---|---|
| A1 翻译存在稳定、可复用、可提炼的失败/成功模式 | 强模型上失败模式消失（zebra×4o 98% 一次解）；弱模型的错误是执行噪声而非知识缺口（注入模板无效，nomem≈baseline） |
| A2 SAT/UNSAT 是可靠的质量门控信号 | AR-LSAT 可判定 SAT 轨迹 77% 答案错误；SAT 门控挖出伪范式 |
| A3 更努力地修复（策略升级）单调改善结果 | 升级阶梯把 54–60 题从诚实失败转为自信错误（wrong-SAT），−9pp |

**从废墟中幸存并升值的资产**：
1. wrong-SAT 现象的量化与机制解剖（baseline=0 vs 记忆系统=54–60，精确等于亏损）；
2. "验证信号必须与任务目标对齐"这一原理的正反两面证据；
3. 轨迹答案验证协议（利用答案空间结构先验做三值判定）——核心思想可前移到**测试时**；
4. 全套可复现的并行实验基础设施与两个基准的干净基线（zebra 50.8%、AR-LSAT 47.5%）。

## 二、研究问题与意义（细化）

### 2.1 现象（论文的第一句话）

LLM+求解器流水线的修复回路以**可满足性**为反馈信号，而任务目标是**正确性**。
当修复被更积极地驱动时，系统学会通过**弱化形式化**来达到 SAT——把"诚实的失败"
（无解，可被发现、可被修复、可被弃答）转换为"自信的错误"（有解但错，对整个
流水线不可见）。我们称之为 **satisfiable-but-wrong（SBW）坍缩**。

SBW 的危害有一个重要的不对称性：UNSAT 失败是**自标注的**（系统知道自己失败了，
可以重试或弃答），SBW 是**静默的**（系统以为自己成功了）。对下游而言，
把 x% 的 UNSAT 失败转换成 x% 的 SBW 不是零和——它摧毁了系统输出的可信度语义。

### 2.2 形式化设定

一个自然语言约束推理实例是三元组 **(P, π, A\*)**：
- P = (背景描述, 线索集 L = {l₁,…,l_m})：自然语言问题；
- π：**答案空间结构先验**——任务定义自带、无需任何标注：
  - 逻辑网格谜题：解唯一，|Sol(C\*)| = 1（谜题生成协议保证）；
  - 五选一分析推理：良定性——恰有一个选项满足题型谓词
    （could-be-true 题恰有一个选项可满足，其余四个不可满足；must/cannot 类似镜像）；
- A\*：真实答案（测试时不可见）。

LLM 翻译 T: P → (C, cov)，其中 C = {c₁,…,c_n} 为形式约束集，
cov ⊆ L × C 为**线索-约束覆盖账本**（每条约束声明它翻译自哪条线索；
由翻译 prompt 强制输出，实现上挂靠 assert_and_track 的追踪名）。
求解器 S 提供可判定查询：SAT?(C)、unsat_core(C)、model(C)、
model_excluding(C, M)（断言 ¬M 后再求解——枚举第二个模型）。

**失败类型学**（按 Sol(C) 与 Sol(C\*) 的关系）：

| 类型 | 解空间特征 | 对 SAT 检查 | 现有文献 |
|---|---|---|---|
| 过约束 | Sol(C) = ∅ | 可见（UNSAT core 定位） | 全部注意力所在（Logic-LM 系、VERGE） |
| **欠约束** | Sol(C) ⊋ Sol(C\*)，\|Sol(C)\| > 1 | **不可见**，SBW 主要来源 | **空白** |
| 误约束 | \|Sol(C)\| = 1 但解错 | 不可见 | 空白（但见 2.3：互锁充分时罕见） |

### 2.3 可见性论证（论文的理论骨架）

**观察（zebra 实证）**：完整翻译下，单条线索的语义错误（方向翻转等）几乎总
与其余线索矛盾——错误以 UNSAT **显形**。这是 baseline wrong-SAT = 0 的原因。

**定义（错误可见性）**：约束集的互锁度越高（每个变量被越多约束共同决定），
局部翻译错误越可能触发全局矛盾而暴露给求解器。

**命题（非正式）**：弱化式修复（删除/放松约束）单调降低互锁度，
把错误从 UNSAT 可见域迁移到 SAT 不可见域。因此：
(i) 修复策略必须**保全可见性**（不净弱化）；
(ii) 系统必须**主动探测不可见域**（用 π 检查解空间形状）。

这两条推论分别对应方法的机制 3 和机制 1/2。

### 2.4 研究问题（分解）

**RQ 主**：不接触真实答案，能否用"结构先验 π + 求解器解空间查询"构造与正确性
对齐的机械验证信号，使 LLM 形式化修复避免 SBW 坍缩并提升端到端正确率？

- **RQ1（现象）**：SBW 坍缩是否真实、可量化、随修复强度增长？
  ——已有数据回答（54–60/640 精确核算 + 77% 轨迹研究），构成论文动机节。
- **RQ2（检测）**：π-gate 对 SBW 的检出能力如何？
  ——度量：以真值离线核算 π-gate 的查准/查全（欠约束检出率、误杀率）；
  预期查全接近 100%（欠约束 ⇒ 多解 ⇒ 必被唯一性检查捕获），
  误杀 = 误约束型 SBW（唯一但错）的占比，预期很小且可量化。
- **RQ3（定位与修复）**：模型差分能否把欠约束定位到线索级，且 LLM 能否据此补全？
  ——度量：差分变量→缺陷线索的定位命中率；补全修复的一次成功率/轮次。
- **RQ4（策略）**：可见性保全不变量能否让"积极修复"安全化？
  ——对照：nopar（积极修复，SBW=54）vs nopar+三机制（预期 SBW→≈0，
  修复收益保留），关键判据是 **净准确率超过 baseline**。
- **RQ5（泛化）**：同一信号原理能否跨答案空间结构实例化？
  ——zebra 的唯一性 ↔ AR-LSAT 的良定性，同一框架两种 π。

### 2.5 意义（对四个社区分别说清楚）

1. **对神经符号推理**：补全反馈信号体系缺失的一半。十年来这条线的修复信号
   全部来自过约束侧（报错、UNSAT、core、MCS）；欠约束侧此前连诊断词汇都没有。
   双向诊断（core ↔ 模型差分）使修复回路第一次对两类翻译缺陷都闭环。
2. **对 LLM 自修复研究**：该文献已知"自我修复会平台化"“LLM 无法自我纠错”
   （Huang et al. 2024），普遍解药是"引入外部验证器"。我们给出一个更精细的教训：
   **外部验证器的信号若与任务目标错位，修复不只是平台化，而是负向坍缩**
   ——且以不可见的方式。这把"要不要外部反馈"的讨论推进到"外部反馈的对齐度"。
3. **对可靠 AI / selective prediction**：π-gate 天然给出弃答语义
   （非唯一 ⇒ 拒绝作答），风险-覆盖曲线是免费副产品。SBW=0 意味着
   系统输出"有答案"时的条件正确率可信——这是部署视角的核心价值。
4. **对约束编程社区**：机制 2 是约束获取（constraint acquisition）经典操作的
   LLM 时代重演——QuAcq 用区分性赋值向人类提成员查询，我们用模型差分向
   "LLM+原文"提查询。这个连接给 CP 会议背景的 AAAI 审稿人一个熟悉的锚点。

## 三、相关工作与背景（按对话关系组织）

> 写作原则：每小节末尾一句话说清"他们做了什么、缺什么、我们补什么"。

### 3.1 LLM+求解器的神经符号推理（本文的宿主谱系）
Logic-LM（Pan et al. 2023）确立翻译-执行-修复三段式；LINC（Olausson et al. 2023）
用 Prover9；SatLM（Ye et al. 2023）总结翻译错误类型；Logic-LM++（Kirtania et al.
2024）成对比较选择修复；Logic.py（Kesseli et al. 2025）用 DSL 降低翻译难度；
VERUS-LM（Callewaert et al. 2025）对接 IDP-Z3；VERGE（Singh et al. 2026）用
最小修正子集（MCS）定位冲突声明。
**共同点：修复信号全部来自不可满足侧**（报错串 / UNSAT / core / MCS）；
执行成功即接受。**缺**：对 SAT-but-wrong 无诊断、无度量、无机制。
**我们**：给出失败类型学、SBW 量化，与 core 对称的欠约束侧信号。
（与 VERGE 需专段对照：MCS 枚举"删哪些约束可恢复 SAT"，仍是过约束工具；
我们的模型差分回答"缺哪些约束才唯一"，方向相反、正交互补。）

### 3.2 LLM 自修复的局限（本文动机的文献基础）
Self-Refine（Madaan et al. 2023）、Reflexion（Shinn et al. 2023）依赖自反馈；
Huang et al.（2024）"LLMs cannot self-correct reasoning yet" 与
Olausson et al.（2024）"Is Self-Repair a Silver Bullet?" 系统性泼冷水；
共识解药是外部验证信号（工具、执行器、求解器）。
**缺**：默认外部信号与目标对齐。**我们**：实证展示错位的外部信号
造成比无信号更糟的静默失败（−9pp、54–60 例自信错误），
并给出对齐方案。这是对"external feedback"叙事的一次必要修正。

### 3.3 弱监督语义解析中的伪程序（SBW 的镜像现象）
Weakly-supervised semantic parsing 早已知道 **spurious programs**：
执行结果碰巧正确但语义错误的程序（Pasupat & Liang 2016; Guu et al. 2017）。
SBW 是其镜像：**求解成功但答案错误的形式化**。他们的解药（一致性、覆盖先验）
启发了我们的结构先验路线。**缺**：该文献处理的是"答案对、程序错"，
监督信号是答案；我们的设定答案不可见，监督信号只能来自答案空间的**形状**。

### 3.4 程序合成与 CEGIS 的区分性输入（机制 2 的学术锚点）
CEGIS / oracle-guided 合成（Solar-Lezama et al.; Jha et al. 2010）用
**两个候选程序的区分性输入**向 oracle 提问以裁剪假设空间。
机制 2 的模型差分同构：两个模型的差异变量就是"区分性赋值"，
oracle 换成"LLM 重读原文线索"。**缺**：CEGIS 需要可执行 oracle；
**我们**证明在 NL 形式化场景，原文本身经覆盖账本索引后可充当弱 oracle。

### 3.5 约束获取（CP 社区的直接前身）
QuAcq / ConAcq（Bessiere et al.）从成员查询中学习缺失约束，
查询对象是人类用户，代价高是其主要瓶颈。
**我们**：查询对象换成 LLM+原文，把约束获取嵌入修复回路——
"欠约束修复 = 自动化的约束获取"是一句能让 CP 审稿人秒懂的定位。

### 3.6 解空间技术（工具层背景）
模型计数（#SAT）、解枚举、MUS/MCS 枚举（Liffiton & Sakallah 2008）、
谜题生成中的唯一性约束（生成器用"恰一解"保证 well-posedness——
这正是 π 的来源，说明结构先验在数据构造侧早已存在，只是从未被
反哺到求解侧）。我们只用其中最廉价的操作（第二模型查询），不依赖完整计数。

### 3.7 Selective prediction / 弃答
Chow (1957) 以降的 reject option；LLM 时代的 abstention 研究。
**我们**：π-gate 的"非唯一即拒答"给神经符号流水线第一个
有形式语义的弃答准则（拒答 ⇔ 解空间形状与先验不符）。

## 四、方法细节（SPARC：三机制 + 状态机）

### 4.0 总体：修复回路从二态到三态

现有流水线的状态机：`UNSAT → 修复；SAT → 接受`。
SPARC 的状态机：

```
           ┌────────────── UNSAT ──────────────┐
           │   core-guided repair（继承，机制3约束）  │
           ▼                                    │
  翻译 T(P) → C, cov ──求解──┤
           │                                    │
           │        SAT ∧ |Sol|>1（π-gate 检出欠约束）
           │                ▼
           │     diff-guided completion（机制2，机制3约束）
           │                │
           └── SAT ∧ 唯一 ∧ 良定 → 接受并作答
                    （预算耗尽 → 弃答，输出"无可信解"）
```

三态修复的每一轮都带预算；终止语义是 selective prediction：
只有通过 π-gate 的答案才输出。

### 4.1 机制 1：结构先验门控（π-gate）

**zebra 实例化（唯一性检查）**
```
输入: C (SAT), 模型 M
1. C' ← C ∪ { ¬(⋀_{v∈vars(M)} v = M[v]) }     # 阻塞子句
2. r ← SAT?(C')
3. r = UNSAT → 唯一 → PASS
   r = SAT   → 取第二模型 M'，返回 (FAIL_UNDER, M, M')   # 进入机制 2
   r = UNKNOWN → 按超时策略降级为 PASS-with-flag（记录，不作答影响待消融）
```
成本：每次接受多一次求解调用（zebra 规模下 <0.2s）。

**AR-LSAT 实例化（良定性检查）**
选项谓词 {φ_A..φ_E} 在 C 上的判定向量必须与题型的良定模式一致：
could-be-true 题要求 **恰一个** φ_k 满足 SAT(C ∧ φ_k)；must-be-true 题要求
恰一个 φ_k 满足 UNSAT(C ∧ ¬φ_k)；EXCEPT 镜像。
判定向量与模式不符（0 个或 ≥2 个候选）⇒ FAIL_UNDER 或 FAIL_OVER
（0 候选多为过约束或选项翻译缺陷；≥2 候选为欠约束——分流进不同修复）。
注：这把我们已验证的轨迹判定逻辑（judge()）从挖掘期前移到测试时。

**误杀分析（需在论文中报告）**：π-gate 唯一无法检出的 SBW 是
"误约束型"（唯一但错）。RQ2 实验用真值离线核算其占比；
可见性论证预言互锁充分时该占比很小（zebra baseline 实测 0/640）。

### 4.2 机制 2：模型差分归因（diff-guided completion）

**输入**：欠约束证据 (M, M')，覆盖账本 cov，线索集 L。

```
1. D ← { v : M[v] ≠ M'[v] }                    # 差异变量集
2. L_D ← { l ∈ L : mentions(l, D) }             # 提及差异变量的线索（词面/实体匹配）
3. 缺陷候选排序：
   a. L 中无覆盖的线索（cov 中无对应约束）且 ∈ L_D   —— 漏译，最高优先
   b. L_D 中覆盖约束为弱形式的线索                    —— 弱译（如 < 译成 ≠）
   c. 其余 L_D                                       —— 语义偏差候选
4. 修复 prompt（一次一条线索）：
   给 LLM：原线索文本 + 该线索现有约束（或"无"）+ D 的两组取值对照
   要求：输出能区分 M 与 M' 的更强约束（必须使二者至少其一违背）
5. 验证接受：新约束 c* 需满足
   a. SAT(C ∪ {c*})                              # 不引入矛盾
   b. ¬(M ⊨ c* ∧ M' ⊨ c*)                        # 真的区分了两个模型（进展保证）
   c. 机制 3 不变量
6. 回到求解 → π-gate；每轮至多 k_diff 次（预算）
```

**进展保证（可证明的小命题）**：步骤 5b 确保每轮接受的约束至少排除一个
现存模型，解空间严格缩小 ⇒ 欠约束修复在 |Sol(C)| 上单调收敛，
不会像 SAT-seeking 修复那样打转。这是论文里为数不多可形式陈述的性质，值得写成命题。

**与 UNSAT core 的对称性（写作卖点）**：
core = "这些约束合谋导致无解"（删/改的候选）；
diff = "这些变量在多解间摇摆"（补/强化的候选）。
一收一放，构成完整的双向诊断。

### 4.3 机制 3：可见性保全不变量（no-weakening invariant）

修复动作 a: C → C' 的准入规则（三条，全部机械可查）：

1. **覆盖不降**：∀l ∈ L，cov 中 l 的关联约束数在 C' 中不少于 C
   （允许替换，禁止裸删除；删除必须伴随同线索的替代翻译）；
2. **解空间单调性（按修复方向）**：
   欠约束修复要求 |Sol(C')| < |Sol(C)|（由 4.2-5b 保证）；
   过约束修复允许放松，但**只允许改动 UNSAT core 内约束**，
   且改后需对该线索做再对齐检查（LLM 判断新约束是否仍忠实于线索原文——
   一次廉价调用，或用词面规则近似）；
3. **SAT 后必过 π-gate**：任何修复达成的 SAT 不再直接接受（4.0 状态机）。

这三条合起来堵死 SBW 通道：规则 1/2 阻止互锁被侵蚀（错误保持可见），
规则 3 兜底探测残余的不可见错误。原 PRISM 的 L1–L4 升级阶梯废除；
如保留温和的目标切换（L1），必须在不变量约束下运行。

### 4.4 覆盖账本（cov）的获得

翻译 prompt 增加一列输出：每条约束前缀 `# clue_i`。
解析后建立 cov；无标注的约束归入 `clue_0`（背景/域约束）。
账本质量本身可度量（漏标率），并在消融中检验其对机制 2 定位命中率的影响。
（工程注记：assert_and_track 的追踪名可直接编码线索号，无新增求解器机制。）

### 4.5 实验设计骨架（对应 RQ，不在本文档展开）

- **主表**：zebra 官方 640 × {Logic-LM 式 baseline, baseline+SPARC,
  积极修复(nopar), 积极修复+SPARC} × 3 种子 × GPT-4o-mini；GPT-4o 对照行。
- **SBW 核算表**：各系统 SAT且对 / SAT但错 / 弃答 三列（论文 A 的签名表格）。
- **风险-覆盖曲线**：π-gate 弃答语义下的 selective prediction 视角。
- **消融**：−π-gate / −模型差分（检出后盲重译）/ −不变量 / −覆盖账本（差分不定位）。
- **RQ2/RQ3 度量表**：检出查准查全、定位命中率、补全成功轮次。
- **AR-LSAT**：良定性实例化重复主表（RQ5）。

**预答辩**：
- "唯一性检查是 folklore" → 单件确实；贡献在完整信号体系 + 差分归因 +
  进展保证 + SBW 诊断证明其必要性。写作上机制 2/3 当技术主角。
- "与 VERGE 区别" → MCS 在过约束侧，我们补欠约束侧，正交（专段对照）。
- "只在弱模型上有效？" → GPT-4o 对照行如实呈现；SBW 是修复回路的结构性质，
  强模型修复少但一旦修复同样面临信号错位（且 π-gate 成本近零，无害保险）。

  对它无增益；论文的胜负手在 RQ4——"积极修复 + SPARC ≥ baseline 且 SBW≈0"。
